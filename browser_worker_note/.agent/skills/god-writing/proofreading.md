# 校正・推敲（note / X 特化）

文章の誤りを発見・修正し、品質を高める最終工程。
「書いてから削る」の「削る」フェーズ。

---

## 校正と推敲の順序

1. **推敲（Editing）** — 構成・表現の改善（先にやる）
2. **校正（Proofreading）** — 誤りの修正（後にやる）

推敲で文を大きく変えた後に校正しないと二度手間になる。

---

## 校正チェック

### 誤字脱字

| 方法 | やり方 |
|------|--------|
| 音読 | 声に出して読むと引っかかる |
| 逆読 | 最後の文から逆に読む（意味を追わず文字を見る） |

よくある誤字: 「以外」⇔「意外」、「移行」⇔「以降」、「対応」⇔「対応」

### 表記揺れ

一つの記事内で統一する。

| チェック対象 | 例 |
|-------------|-----|
| 漢字/ひらがな | 「事」⇔「こと」「時」⇔「とき」 |
| 外来語 | 「アドイン」⇔「アドオン」 |
| 数字 | 「3つ」⇔「三つ」 |
| 記号 | 半角「!」⇔全角「！」 |

### このアカウントの表記基準

| 項目 | 統一ルール |
|------|-----------|
| 数字 | 算用数字（「3つ」「87%」） |
| AI名 | ChatGPT, Claude, Gemini, Grok（公式表記に準拠） |
| 「こと」「とき」 | ひらがな |
| 記号 | 全角句読点「。」「、」。感嘆符は全角「！」「？」 |

### 文法チェック

| チェック | よくあるミス |
|---------|------------|
| 主述の対応 | 主語と述語が噛み合わない |
| 係り受け | 修飾語が遠い被修飾語にかかる |
| 「たり〜たり」 | 片方だけ「たり」 |
| 二重敬語 | 「おっしゃられる」等 |

## 推敲チェック

### 論理チェック

| チェック | 確認方法 |
|---------|---------|
| 矛盾 | 前後で相反する記述がないか |
| 飛躍 | 根拠なしに結論に至っていないか |
| 一貫性 | 主張が記事全体で統一されているか |

### ファクトチェック（このアカウント固有）

→ 詳細は `knowledge/account-concept.md` のファクトチェックポリシー参照

| 対象 | 確認先 |
|------|--------|
| AI機能・性能 | 公式ドキュメント or リリースノート |
| 数字 | 出典明記。出典不明の数字は使わない |
| 料金・プラン | 公式ページ + 「執筆時点」明記 |
| キャラの発言 | AIの実際の機能に基づいているか |

### 読みやすさチェック

| チェック | 基準 |
|---------|------|
| 文長 | 60文字以内（X投稿は40文字） |
| 段落長 | 3〜5行（スマホで2〜3行） |
| 文字の壁 | 300文字以上の連続テキストがないか |
| 冗長表現 | concise-writing のチェックリスト参照 |

---

## 推敲のプロセス

1. 全体通読 — 流れ・論理・構成を確認
2. 見出しだけ読む — スキャンで内容が伝わるか
3. 段落ごと — トピックセンテンス・一段落一主題
4. 文ごと — 文法・表現・文長
5. 表記統一 — 検索で確認
6. 音読 — リズム・違和感
7. 10-20%削減 — 冗長表現の最終カット

---

## このアカウント固有の最終チェック

| チェック | 基準 |
|---------|------|
| 驚き屋ワード | 「やばい」「神」「革命」がないか |
| 前置き | 「正直に言います」等のメタコメンタリーがないか |
| トーン | 「先輩が見せてくれる感じ」になっているか |
| Level 4以上 | Qiita/Zennと同じ品質（Level 3）になっていないか |
| CTA | 「すごい」で終わらず「やってみよう」で着地しているか |
| 制限事項 | デメリット・制限を隠していないか |

## チェックリスト

- [ ] 誤字脱字はないか（音読で確認）
- [ ] 表記は統一されているか
- [ ] 文法エラーはないか（主述・係り受け）
- [ ] 事実は正確か（出典確認済み）
- [ ] 論理に矛盾・飛躍はないか
- [ ] 一文60文字以内か
- [ ] 冗長表現は削除したか（10-20%削減）
- [ ] 驚き屋ワード・前置きがないか
- [ ] Level 4以上の品質か
