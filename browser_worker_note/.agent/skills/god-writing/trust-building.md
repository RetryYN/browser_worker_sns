# 信頼構築（note / X 特化）

文章で信頼を構築する技法。
NOT Believe を突破し、読者に「この人は信頼できる」と思わせるスキル。

---

## 信頼の4要素

4つが揃って初めて信頼が生まれる。1つでも欠けると崩れる。

| 要素 | 意味 | このアカウントでの実現 |
|------|------|----------------------|
| 能力（Competence） | 専門知識がある | 実画面スクショ、一次ソース、具体的な操作手順 |
| 誠実さ（Integrity） | 正直で倫理的 | デメリットも書く、驚き屋ムーブしない |
| 善意（Benevolence） | 読者の利益を考えている | 「やってみよう」で着地、売りつけない |
| 一貫性（Consistency） | 言行が一致 | トーン・スタンス・品質を維持 |

---

## 専門性の示し方

このアカウントに肩書き・受賞歴・著書はない。
間接的な方法で「この人は詳しい」と感じさせる。

### 使える手法

| 手法 | このアカウントでの形 |
|------|---------------------|
| 深い知識に基づく解説 | 公式ドキュメントまで遡った説明 |
| 専門用語の正確な使用 + 翻訳 | 「MCP（AIがツールに接続する仕組み）」 |
| 具体的で詳細な情報 | ベンチマーク数値 + 出典明記 |
| 質の高いコンテンツ | 実画面・図解・掛け合いの三位一体 |

### 使えない手法（このアカウントに不適）

- 資格・経歴・肩書きの提示
- 受賞歴・メディア掲載
- 著書・論文の引用
- 有名企業のロゴ

→ 代わりに**コンテンツの質そのもの**で専門性を示す。

---

## 誠実さのシグナル

完璧すぎる主張は逆に疑われる。ミサキ層は驚き屋の誇張に慣れている。

### このアカウントでの実践

```
OK: 「ただし日本ではまだ使えません」
→ デメリットを隠さない

OK: 「精度87%（OpenAI公式発表）。実際に使ってみないと体感はわからない」
→ 不確実性を認める

OK: 「前回の記事で〇〇と書きましたが、公式発表で修正がありました」
→ 間違いを認める

NG: 「革命的な機能で全てが変わる！」
→ 驚き屋ムーブ = 信頼崩壊
```

### デメリット開示のパターン

| パターン | 例 |
|---------|-----|
| 地域制限 | 「現在は米・加・豪のみ。日本は今後数ヶ月で拡大予定」 |
| 料金制限 | 「Plus以上の有料プランが必要」 |
| 機能制限 | 「Excel Online限定。デスクトップ版は未対応」 |
| 精度の限界 | 「複雑な数式では期待通りにいかないこともある」 |

## 社会的証明の使い方

→ 詳細は `social-proof.md` を参照

このアカウントで使えるのは:
- **体験の証明**: 実画面スクショ（最強）
- **類似者の証明**: ミサキ層の「あるある」共感
- **キャラの証明**: AIが自分の機能として語る

---

## 透明性の確保

### このアカウントで開示すべき情報

| 情報 | 開示方法 |
|------|---------|
| 情報源・出典 | 記事末尾に公式ソースリンク |
| 執筆時点 | 「2026年3月時点の情報です」 |
| AI活用 | このアカウント自体がAIで運営されていることは裏側回で公開 |
| 未検証情報 | 「〜と発表されています（執筆時点で未検証）」 |

### 不要な開示

- アフィリエイト開示（アフィリエイトをやっていない）
- 利害関係の開示（特定企業との関係はない）
- 個人情報（匿名アカウント）

---

## 一貫性の構築

### メッセージの一貫性

| 一貫すべきこと | 具体的なルール |
|---------------|--------------|
| スタンス | 驚き屋のアンチテーゼ。正直に見せる |
| トーン | 先輩が隣で見せてくれる感じ |
| 品質 | Level 4以上。Level 3（技術ブログ同等）は不合格 |
| 着地点 | 常に「やってみよう」。「すごい」で終わらない |

### 一貫性を壊すもの

```
NG: ある記事では「驚き屋ダメ」、別の記事では「やばい！神！」
→ スタンスのブレ = 信頼崩壊

NG: ある記事は実画面つき、別の記事はテキストだけ
→ 品質のブレ = 「手抜きか？」

NG: ある記事では制限を正直に、別の記事ではメリットだけ
→ 誠実さのブレ = 驚き屋と同じ
```

## 権威の構築

自称の権威は使えない。**コンテンツと引用で権威を借りる**。

| 手法 | このアカウントでの形 |
|------|---------------------|
| 一次ソース引用 | 「OpenAI公式ブログによると」 |
| データ引用 | 「財務モデリングベンチマーク87.3%（OpenAI発表）」 |
| キャラの証言 | ChatGPTが「自分の機能」として語る |
| 実画面 | Playwright MCPで撮った本物の画面 |

---

## 信頼を破壊する要素（絶対NG）

| 破壊要素 | このアカウントでの注意 |
|---------|---------------------|
| 誇大表現 | 「絶対」「100%」「必ず」「革命」の多用 |
| 非現実的な約束 | 「これで仕事が全部自動化される」 |
| 情報の隠蔽 | 日本未提供なのに触れない |
| 驚き屋ムーブ | 「やばい！神！」で感情だけ煽る |
| 出典なし | 数字を断定するが根拠を示さない |
| 使っていないのに「使ってみた」 | 体験の偽装 = 発覚時に致命傷 |

**一度失った信頼の回復は非常に困難。** 10記事で積んだ信頼が1記事で崩れる。

---

## 記事内での配置

```
リード ────── 誠実さシグナル（制限・条件を冒頭で正直に）
本文前半 ──── 専門性の証明（一次ソース・具体的数字）
本文中盤 ──── 体験の証明（実画面スクショ・操作結果）
掛け合い ──── キャラの証言（AIが自分の機能を語る）+ デメリット代弁
CTA ──────── 善意の証明（読者の利益を考えた行動提案）
末尾 ──────── 一貫性の維持（次回予告で期待に応える）
```
