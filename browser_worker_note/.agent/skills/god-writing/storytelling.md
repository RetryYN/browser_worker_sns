# ストーリーテリング（note / X 特化）

物語の力で読者を引き込むスキル。
このアカウントのストーリーは「個人の体験談」ではなく「キャラの掛け合い」と「読者の変化」。

---

## このアカウントのストーリー素材

個人の創業物語やサクセスストーリーは持っていない。
代わりに使えるストーリー要素:

| 素材 | 使い方 |
|------|--------|
| キャラの掛け合い | AIの個性をぶつけて笑いと学びを生む |
| 読者の「あるある」 | ミサキ層の日常をシーンで描写 |
| Before/After | 手作業 → AI活用の変化を見せる |
| 「やってみた」の経緯 | 試行 → 結果 → 発見の流れ |

---

## 掛け合いストーリーの構造

### 基本パターン: 問題提起 → 各キャラの反応 → ツッコミ → 学び

```
アイコ「Excel関数、毎回調べるのつらくない？」（問題提起 = 読者共感）
ChatGPT「任せてくれたら5秒で終わるけど」（解決の提示）
Claude「私もファイル分析は得意ですよ」（対抗 = 選択肢の提示）
Gemini「あ、それ知ってます！確か〜」（ボケ = 場の緩和）
アイコ「話を戻していい？」（ツッコミ = 読者の代弁）
```

### 掛け合いの役割

| 役割 | 効果 |
|------|------|
| 読者の疑問を代弁 | アイコが「本当に？」と聞く = 読者が聞きたいこと |
| 複数の視点を提示 | ChatGPT vs Claude = 読者の選択肢 |
| 難しい話を崩す | Geminiのボケで緊張を解く |
| 情報の記憶定着 | キャラの口癖と結びつけて覚えやすい |

---

## 読者の変化のストーリー

記事全体が「読者の小さな変化」の物語になっている。

```
Before: 「Excel関数わからない。毎回ググる。つらい」
  ↓ 記事を読む
Discovery: 「ChatGPTに聞けばいいのか」「日本語で指示するだけ？」
  ↓
After: 「やってみよう」
```

この変化を設計するのがストーリーテリングの仕事。

---

## マイクロストーリー（1〜3文）

長い物語は不要。1〜3文の超短編で十分。

```
「先週、ChatGPTに「B列の合計出して」と言ったら、
 SUM関数が3秒で出てきた。VLOOKUPより簡単だった。」
→ 体験の断片。信頼 + 具体性

「チームの後輩がChatGPTでレポート作ったら、
 先輩に「どうやったの？」と聞かれたらしい。」
→ 類似者の証明。「自分も」と思わせる
```

---

## ストーリーの注意点

| NG | 理由 |
|----|------|
| 「3年前の私は〜」 | このアカウントに個人の物語はない |
| 長すぎるストーリー | ミサキ層は離脱する。3〜5往復で切る |
| 嘘の体験談 | 「使ってみた」と書いて使っていない = 信頼崩壊 |
| 掛け合いに情報を混ぜる | 情報パートと掛け合いは分離する |
| オチなし | 掛け合いは必ずアイコのツッコミで締める |

---

## 記事内での配置

```
冒頭 ────── 掛け合い（読者の「あるある」で入る）or リード直入り
本文 ────── 情報パート（説明ベース。ストーリーではない）
ブレイク ── 掛け合い（300文字ルールで崩す。1記事1-2箇所）
CTA ────── マイクロストーリー or 行動提案
```
