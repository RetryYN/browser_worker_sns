# 簡潔ライティング（note / X 特化）

無駄を省き、読者の時間を尊重する。
god-writing 原則3「価値密度最大化」の実践スキル。

---

## このアカウントでの簡潔さの意味

ミサキ層はスキャンで読む。一文が長いだけで離脱する。
簡潔さは「冷たさ」ではなく「優しさ」。

| 媒体 | 一文の目安 | 段落の目安 |
|------|-----------|-----------|
| note記事 | 60文字以内 | 3〜5行 |
| X投稿 | 40文字以内 | 2〜3行 |
| 掛け合い | 30文字以内 | 1〜2行 |

---

## 削るべき冗長表現

### パターン1: 形式表現

| 冗長（NG） | 簡潔（OK） |
|-----------|-----------|
| 〜することができる | 〜できる |
| 〜を行う | 〜する |
| 〜に関して | 〜は / 〜について |
| 〜であるということ | 〜だ |
| 〜しなければならない | 〜が必要 |
| 〜するためには | 〜には |
| 〜という | （削除可能な場合が多い） |

### パターン2: 弱い動詞

| 弱い（NG） | 強い（OK） |
|-----------|-----------|
| 確認を行う | 確認する / 調べる |
| 検討を加える | 考える |
| 対応を実施する | 対応する |
| 分析を行った結果 | 分析した結果 |

### パターン3: 不要な修飾語

| 不要（NG） | 対応 |
|-----------|------|
| 「非常に便利」 | 便利。またはどう便利か具体的に |
| 「様々な機能」 | 4つの機能（数字で具体化） |
| 「基本的に」「原則として」 | 曖昧なら削除 |
| 「ある意味では」「いわば」 | 自信がないなら主張自体を見直す |

### パターン4: 重複

| 重複（NG） | 簡潔（OK） |
|-----------|-----------|
| 今後の将来において | 将来 |
| 一番最初に | 最初に |
| まず最初に | まず |
| 約〜ぐらい | 約〜 / 〜ぐらい |

## このアカウント固有の簡潔化ルール

### 専門用語の翻訳は残す

```
NG: 「MCP経由で連携」
→ ミサキ層に伝わらない

OK: 「MCP（AIがツールに接続する仕組み）経由で連携」
→ 翻訳は冗長ではない。読者のための情報

NG: 「MCP（Model Context Protocol）というのはAIがツールに接続するための仕組みであり、これによって〜」
→ 翻訳のあとの説明が冗長
```

### 掛け合いは短く切る

```
NG:
ChatGPT「Excelの数式を自動生成する機能については、私が最も得意としているところですので、安心して任せてください」

OK:
ChatGPT「Excel？ 数式も分析も任せて」
```

### 情報パートは一文一義

```
NG:
「ChatGPT for Excelは、OpenAIが3月5日に発表したExcelアドインで、自然言語で数式を作成したりエラーを修正したりシナリオ分析を行ったりすることができる新機能です。」

OK:
「ChatGPT for Excelが発表されました。
 Excelの中でAIに指示できるアドインです。
 数式の作成やエラー修正をやってくれます。」
```

## 一語への置き換え

| 長い表現 | 短い表現 |
|---------|---------|
| 〜である場合には | 〜なら |
| 〜することによって | 〜で / 〜して |
| 〜と思います | 〜です（断定可能なら） |
| 検討を行った結果として | 検討の結果 |
| 確認することが必要です | 確認が必要です |

---

## 情報の取捨選択

全てを書く必要はない。

| 優先度 | 扱い | 判断基準 |
|--------|------|---------|
| 必須 | 本文に書く | これがないと記事の目的を達成できない |
| 補足 | 図解 or 掛け合いで補う | あると理解が深まるが、なくても伝わる |
| 余剰 | 削除 | 「この情報がないと読者は困るか？」→ No |

### このアカウントでの判断例

```
必須: 「何ができるか」「日本で使えるか」「今日やれること」
補足: ベンチマーク数値、前世代との比較、料金体系
余剰: 技術的な仕組みの詳細、API仕様、開発者向け情報
```

---

## 編集プロセス

1. まず書きたいことを全て書く
2. 10〜20%の文字数削減を目標に削る
3. 一文60文字以内か確認（X投稿は40文字）
4. 削除して意味が変わらないか確認
5. 声に出して読み、リズムを確認

**「一度書いてから削る」** が鉄則。最初から簡潔に書こうとすると詰まる。

---

## 簡潔すぎの注意

| 削りすぎのサイン | 対応 |
|----------------|------|
| ミサキ層に伝わらない | 翻訳・具体例を戻す |
| 冷たい印象 | 掛け合いパートで温かみを補う |
| 根拠が不足 | 出典・数字を戻す |

**原則: 情報パートは簡潔に。温かみは掛け合い・キャラで出す。**
