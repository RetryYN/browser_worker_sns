# 文構築（note / X 特化）

一文を明確で伝わりやすく構築する技法。
全てのライティングスキルの土台。

---

## 一文一義

一つの文に一つの情報だけ入れる。

```
NG:
「ChatGPT for Excelは、OpenAIが3月5日に発表したExcelアドインで、
 自然言語で数式を作成したりエラーを修正したりすることができる新機能です。」
→ 3つの情報が1文に詰まっている

OK:
「ChatGPT for Excelが発表されました。
 Excelの中でAIに指示できるアドインです。
 数式の作成やエラー修正をやってくれます。」
→ 1文1情報
```

---

## 文長の基準

| 媒体 | 目安 | 最大 |
|------|------|------|
| note記事（情報パート） | 40字 | 60字 |
| X投稿 | 30字 | 40字 |
| 掛け合い | 20字 | 30字 |

**確認方法:** 声に出して一息で読めるか。

---

## 主語の扱い

日本語は主語省略が自然。ただし行為者が重要なときは明示。

```
省略OK:
「数式を自動生成します。」
→ 文脈でChatGPT for Excelとわかる

省略NG:
「検討した結果、導入を決定しました。」
→ 誰が検討した？ 誰が決定した？

OK:
「OpenAIが発表しました。」
→ 行為者が重要な情報
```

---

## 述語を早く出す

日本語は述語が文末。長い文では結論が遅れる。

```
NG:
「昨日OpenAIが公式ブログで発表した内容によると、
 GPT-5.4を搭載したExcelアドインが提供開始されました。」
→ 結論が遠い

OK:
「Excelアドインが提供開始されました。
 OpenAIが昨日発表したGPT-5.4搭載です。」
→ 結論先行
```

---

## 修飾語の配置

**原則: 長い修飾語 → 短い修飾語 → 被修飾語**

```
NG: 「新しい昨日発表された機能」
OK: 「昨日発表された新しい機能」

NG: 「便利なExcelの中で動くアドイン」
OK: 「Excelの中で動く便利なアドイン」
```

修飾語と被修飾語は近づける。間に他の要素を挟まない。

---

## 曖昧さの解消

```
曖昧: 「全ての機能が使えるわけではない」
→ 一部使えない？ 全部使えない？

明確: 「4つの機能のうち、日本で使えるのは2つだけです」

曖昧: 「ChatGPTとClaudeの新しい機能」
→ 両方新しい？ Claudeだけ新しい？

明確: 「ChatGPTの新機能とClaudeの新機能」
```

---

## 並列構造の統一

並列する要素は同じ形式で揃える。

```
NG: 「数式の作成と、エラーを修正すること」
→ 名詞と動名詞が混在

OK: 「数式の作成とエラーの修正」
OK: 「数式を作成し、エラーを修正する」
```

---

## 能動態を基本とする

```
受動態: 「ベータ版が提供開始された」
能動態: 「OpenAIがベータ版を提供開始した」

受動態を使う場面:
- 行為者が重要でない: 「数式が自動生成される」
- 読者目線: 「指示するだけで作ってもらえる」
```

---

## 文のリズム

同じ長さ・同じ文末が続くと単調。変化をつける。

```
NG（単調）:
「ChatGPT for Excelが出ました。
 Excelの中でAIが動きます。
 数式を自動生成してくれます。
 ただし日本では使えません。」
→ 全部同じ長さで「〜ます」

OK（リズムあり）:
「ChatGPT for Excelが出ました。
 Excelの中でAIが動きます。
 セルを選んで日本語で指示するだけで、
 数式を自動生成してくれるアドインです。
 ただし、日本ではまだ。」
→ 短→短→やや長→短。文末も変化
```

---

## 文型パターン

| パターン | 使いどころ | 例 |
|---------|-----------|-----|
| 断定 | 事実の提示 | 「精度は87%です」 |
| 提案 | CTA | 「まず試してみてください」 |
| 疑問 | フック・見出し | 「毎回ググってない？」 |
| 条件 | 手順説明 | 「日本語で指示すれば、数式を作れます」 |
| 対比 | 差別化 | 「調べるのではなく、聞くだけ」 |
