# ベネフィットライティング（note / X 特化）

機能をベネフィットに変換し、読者が本当に欲しい価値を言語化するスキル。
「だから何？（So What?）」を繰り返して深掘りする。

---

## 機能 → ベネフィット変換

### 変換の公式

```
機能（スペック） → だから何？ → だから何？ → ベネフィット（読者の価値）
```

### このアカウントでの変換例

```
機能: 「自然言語で数式を自動生成」
→ だから何？ 「関数を知らなくても数式が作れる」
→ だから何？ 「毎回VLOOKUPをググる時間がなくなる」
→ だから何？ 「月100分の時間が浮く。昼休みが5分長くなる」
                 ↑ ミサキ層に刺さるベネフィット

機能: 「財務モデリングベンチマーク87%」
→ だから何？ 「AIが作る数式の精度が高い」
→ だから何？ 「間違いを自分で直す手間が減る」
→ だから何？ 「AIに任せて安心できる」
                 ↑ 感情的ベネフィット
```

---

## ベネフィットの階層

深い階層ほど読者の心を動かす。

| 階層 | 意味 | ミサキ層での例 |
|------|------|-------------|
| 機能的 | 実用的な利点 | 「数式を自動で作れる」 |
| 感情的 | 感情の変化 | 「Excelで悩まなくなる」 |
| 社会的 | 他者との関係 | 「チームの先輩に驚かれる」 |
| 自己実現 | 理想の自分 | 「AIを使いこなしてる側になる」 |

### このアカウントでの深掘り度

```
情報パート: 機能的ベネフィットまで
→ 「数式を日本語で指示するだけで作れます」

掛け合い: 感情的・社会的ベネフィットまで
→ アイコ「毎回VLOOKUPググるのつらくない？」
→ ChatGPT「任せてくれれば5秒だけど」

CTA: 機能的 + 感情的を組み合わせ
→ 「まず聞いてみてください。5分で『これ楽だ』がわかります」
```

---

## 機能止まりのNG例

```
NG（機能の羅列）:
「ChatGPT for Excelの主な機能:
 ・自然言語数式生成
 ・エラー修正
 ・シナリオ分析
 ・財務モデリング」
→ スペックシート。ミサキ層は「で、何が嬉しいの？」

OK（ベネフィットに変換）:
「ChatGPT for Excelでできること:
 ・関数を知らなくても数式が作れる
 ・「なんかエラー出た」を日本語で直せる
 ・「もし売上が10%上がったら」をAIがシミュレーション
 ・決算レポートの分析をAIに投げられる」
→ 読者の行動シーンで語る
```

---

## ミサキ層に響くベネフィット表現

| 響く（身体感覚） | 響かない（スペック） |
|----------------|-------------------|
| 「毎回ググる時間がなくなる」 | 「自然言語処理ベースの数式生成」 |
| 「5分かかってた作業が30秒」 | 「ベンチマーク87%の精度」 |
| 「日本語で聞くだけ」 | 「マルチモーダル入力対応」 |
| 「エラーの意味がわかる」 | 「エラーハンドリング機能」 |

### 表現パターン

```
「〇〇がなくなる」: 「毎回ググる時間がなくなる」
「〇〇だけで」: 「日本語で聞くだけで数式が作れる」
「〇〇 → △△に」: 「5分 → 30秒に」
「もう〇〇しなくていい」: 「もう関数を暗記しなくていい」
```

---

## Before/Afterでの表現

変化を対比で見せる。このアカウントの武器。

```
Before: 「VLOOKUP 書き方」でググる → 解説記事を読む → コピペする → エラー出る → またググる（5分）
After: ChatGPTに「B列からC列を探して」と言う → 数式が出る（30秒）
```

**具体的なシーンで描写するほど効果的。** 抽象的な「便利になる」は弱い。

---

## ネガティブベネフィット（損失回避）の使い方

得るより失う恐怖は2倍強い。ただしこのアカウントでは抑えめに。

```
OK（ソフト）:
「知っておけば、同僚がAIで作業してるのを見ても焦らない」
→ 小さな損失回避。煽りではない

NG（煽り）:
「今すぐ始めないと取り残されます」
→ 恐怖煽り。驚き屋ムーブ

NG:
「AIを使えない人は淘汰される」
→ NGトーンに抵触
```

## 記事内での配置

```
タイトル ──── 最大のベネフィット（「Excelの関数、もう調べなくていい」）
リード ────── 機能的ベネフィットの要約
見出し ────── 各セクションのベネフィット（「何ができるか」ではなく「何が嬉しいか」）
本文 ──────── 機能 → ベネフィット変換
図解 ──────── Before/Afterの可視化
掛け合い ──── 感情的ベネフィットをキャラが代弁
CTA ──────── 「今日やれること」で機能的ベネフィットを体験させる
```
