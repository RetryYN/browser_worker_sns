# 3NOT理論（note / X 特化）

**マクスウェル・サックハイムの3原則。全ての読者が持つ心理的障壁。**
これを突破しなければ文章は機能しない。

```
読まない → 信じない → 行動しない
(Not Read)   (Not Believe)   (Not Act)
```

---

## NOT 1: 読まない（Not Read）

### 現実

読者はあなたの記事を読まない。スクロールし、飛ばし、離脱する。
noteのタイムラインでサムネイルが目に入っても、タイトルを読まずに通り過ぎる。
タイトルを読んでも、リードの3行で離脱する。読むことは認知コストがかかる。

### ミサキ層の「読まない」

- Xのタイムラインは1秒で流れる。止まる理由がなければ存在しない
- noteの記事は「長そう」と思った瞬間に閉じる
- 見出しだけ飛ばし読みして、刺さらなければ戻ってこない
- 文字の壁（300文字以上の連続テキスト）で離脱する

### 突破策

| 要素 | note記事 | X投稿 |
|------|---------|-------|
| タイトル | 読者のペインか好奇心を起点。28〜32文字 | 1行目で止める。共感 or 意外性 |
| 冒頭 | 3行で全結論。「何がわかるか」を先出し | 本文140文字で価値が完結 |
| 見出し | スキャンで内容が把握できる設計 | — |
| 視覚 | 図解・キャラ画像で崩す。文字の壁を作らない | 画像必須。テキストだけの投稿は避ける |
| 文長 | 一文60文字以内 | 一文40文字以内 |

### 冒頭パターン（note記事）

```
パターンA: ペイン起点
「〇〇で困ってませんか。△△を使えば解決できます。ただし□□です。」

パターンB: 事実起点（ニュース）
「〇〇が発表されました。できることは△△。ただし日本では□□です。」

パターンC: 結果起点
「〇〇をやってみました。結果は△△。やり方を共有します。」
```

共通ルール: **リードに全結論を入れる。** 結論を隠して引っ張らない。

## NOT 2: 信じない（Not Believe）

### 現実

読者はあなたの主張を信じない。Xは驚き屋と誇大広告で溢れている。
「AIすごい」系の投稿に読者は免疫がある。「また煽りでしょ」と思っている。

### ミサキ層の「信じない」

- 驚き屋の「やばい！神！」に慣れて冷めている
- 「本当にそんなことできるの？」が常に頭にある
- 使ったことないツールの紹介は広告に見える
- スペックや数字を並べられても実感がわかない

### 突破策

| 手法 | このアカウントでの実践 |
|------|----------------------|
| 証拠 | **実画面スクショ**。Playwright MCPで撮れるのが最大の武器 |
| 具体性 | 「便利です」ではなく「VLOOKUPを30秒で書いてくれた」 |
| 透明性 | 制限・デメリットを隠さない。「日本では使えない」を正直に書く |
| 権威 | 一次ソース（公式ドキュメント）を明示する |
| 体験 | 「使ってみたらこうだった」。成功も失敗も見せる |

### 信頼度の公式

```
信頼度 = (一次ソース × 実画面 × 透明性) ÷ 驚き屋っぽさ
```

**やってはいけないこと:**
- 使っていないのに「使ってみた」と書く
- メリットだけ並べてデメリットを隠す
- 「やばい」「神」「革命」で感情を煽る
- 出典なしで数字を断定する

### 証拠提示パターン（note記事）

```
パターンA: 実演ベース
「実際に〇〇してみました（スクショ）。結果は△△でした。ただし□□の制限があります。」

パターンB: ソースベース（実演不可の場合）
「公式ドキュメントによると〇〇です。△△の条件が必要です。日本対応は□□の予定です。」
```

## NOT 3: 行動しない（Not Act）

### 現実

読者は納得しても行動しない。「へーすごい」で閉じて、二度と戻ってこない。
「あとでやろう」は「やらない」と同義。

### ミサキ層の「行動しない」

- 「面白かった」でスキ押して終わり
- 「やってみよう」と思っても手順がわからなくて止まる
- ツールのURLがなくて自分で検索する気力がない
- 3ステップ以上あると「めんどくさい」で離脱する

### 突破策

| 手法 | このアカウントでの実践 |
|------|----------------------|
| 明確なCTA | 「次にやること」を1つだけ示す |
| 簡易化 | 手順は3ステップ以内。URLも添える |
| 即時性 | 「今日の昼休みにできること」レベルまで落とす |
| 損失回避 | 「知らないと損」ではなく「知ってると楽になる」 |
| 継続接点 | 「次回は実画面レポート」で期待を残す |

### CTA設計パターン

```
パターンA: 行動促進（ノウハウ記事）
「まずは〇〇を開いて△△と入力してみてください。それだけで□□がわかります。」

パターンB: 期待形成（ニュース記事）
「〇〇が日本に来たら実画面つきでレポートします。試してほしい業務があればコメントで教えてください。」

パターンC: 参加誘導（X投稿）
「あなたの〇〇はどうですか？ リプで教えてください。」
```

**CTAの禁じ手:**
- 「すごい」で終わらせる（行動ゼロ）
- 「ぜひお試しください」（何を試すか不明）
- 「いかがでしたか？」（価値ゼロの定型文）

## 3NOT統合フレームワーク

記事全体での配置:

```
タイトル ──── NOT Read突破（止まる理由を作る）
リード ────── NOT Read突破（3行で全結論）
サムネ ────── NOT Read突破（図解 or キャラで視覚フック）
  │
本文 ──────── NOT Believe突破（実画面・ソース・透明性）
図解 ──────── NOT Believe突破 + NOT Read突破（証拠 + 視覚）
掛け合い ──── NOT Believe突破（キャラが読者の疑問を代弁）
  │
CTA ──────── NOT Act突破（今日やれることを具体的に）
末尾 ──────── NOT Act突破（次回予告で継続接点）
```

## Before/After 例（このアカウント用）

### Before（Level 2: QiitaやZennと同じ）

```
タイトル: ChatGPT for Excelが発表。機能まとめ
リード: OpenAIが新機能を発表しました。詳細を解説します。
本文: 機能一覧を4つ紹介。制限事項の表。代替手段3つ。
CTA: ぜひお試しください。
```

→ 読者: 「公式ドキュメント読めばよくない？」

### After（Level 4: このアカウントらしい）

```
タイトル: Excelの関数、毎回ググってない？ もうすぐAIがやってくれる
リード: Excelで関数を調べる時間、なくなります。〇〇が発表されました。ただし日本はまだ。今すぐできる方法も紹介します。
本文: 読者のペインから入る → 何が変わるか → 制限も正直に → 今日からできること（具体手順）
図解: キャラ入りで「コピペ運用あるある」を可視化
掛け合い: ChatGPTとClaudeが自分の得意分野を主張。アイコがツッコミ
CTA: 「まずChatGPTにExcelファイルを渡してみてください。『B列の合計を出して』だけでOKです」
```

→ 読者: 「やってみよう」

## チェックリスト

### NOT Read
- [ ] タイトルで「自分に関係ある」と思わせているか
- [ ] リード3行で全結論が伝わるか
- [ ] 見出しだけスキャンしても価値が伝わるか
- [ ] 図解・画像が文字の壁を崩しているか
- [ ] 300文字以上の連続テキストがないか

### NOT Believe
- [ ] 一次ソースを提示しているか
- [ ] 実画面スクショ or 具体的な操作結果があるか
- [ ] 制限事項・デメリットを隠していないか
- [ ] 「使ってみた」と書いて使っていない嘘はないか
- [ ] 驚き屋ワード（やばい・神・革命）を使っていないか

### NOT Act
- [ ] 「今日やれること」が具体的に書かれているか
- [ ] 手順は3ステップ以内か
- [ ] CTAが読者の次の行動を1つだけ示しているか
- [ ] 「すごい」で終わらず「やってみよう」で着地しているか
- [ ] 次回予告で継続接点を作っているか
