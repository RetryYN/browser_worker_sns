# 段落設計（note / X 特化）

文章全体の流れを作る中間構造の設計スキル。
スキャンで読むミサキ層に「拾い読みでも伝わる」段落を作る。

---

## 段落の基本構造

| 要素 | 役割 | noteでの実践 |
|------|------|-------------|
| トピックセンテンス | 段落の主題を冒頭で明示 | 見出しを補強する一文目 |
| 支持文 | 主題を説明・補強 | 具体例・数字・出典 |
| 結論文 | まとめ・次への橋渡し | 省略可。短い段落では不要 |

**一段落一主題が原則。** 主題が変わったら段落を変える。

---

## noteでの段落ルール

| ルール | 目安 | 理由 |
|--------|------|------|
| 段落の長さ | 3〜5行（スマホで2〜3行） | 画面の半分を埋めると圧迫感 |
| 文字数 | 100〜200字 | 300字超えると文字の壁（readability-optimization参照） |
| 一文の長さ | 60文字以内 | スキャンで拾える限界 |

### 長すぎる段落の分割

```
NG（1段落に2主題）:
「ChatGPT for Excelは自然言語で数式を作れるアドインです。
 精度は87%で前世代から大幅に向上しました。
 ただし現在は米・加・豪の3カ国限定で、日本は未対応です。
 代わりに今のChatGPTでもExcelファイルを渡せば分析できます。」

OK（2段落に分割）:
「ChatGPT for Excelは自然言語で数式を作れるアドインです。
 精度は87%で前世代から大幅に向上しました。

 ただし現在は米・加・豪の3カ国限定。日本は未対応です。
 代わりに今のChatGPTでもExcelファイルを渡せば分析できます。」
→ 「機能」と「制限+代替」で主題が異なる
```

## トピックセンテンスの書き方

段落の冒頭に置く。スキャンでトピックセンテンスだけ読んでも記事の要点が伝わる設計。

```
OK: 「できることは4つです。」
→ 読者の脳が受け入れ準備をする（チャンキング）

OK: 「日本ではまだ使えません。」
→ 一文で段落の主題が明確

NG: 「ここからは少し残念なお知らせです。」
→ 前置き。何の話か不明
```

## 段落間の接続

### このアカウントで使う接続パターン

| パターン | 使いどころ | 例 |
|---------|-----------|-----|
| 逆接 | 制限・デメリットの提示 | 「ただし」「ただ」 |
| 追加 | 機能の列挙 | 「さらに」「もう一つ」 |
| 転換 | 話題の切り替え | 見出しで区切る（接続詞不要） |
| 因果 | 理由の説明 | 「なぜなら」「その理由は」 |

### 接続の注意

```
NG: 「しかし」「また」「さらに」の連発
→ 単調。見出しで区切るか、段落の順序を見直す

OK: 見出しで大きく区切り、段落内は接続詞最小限
→ noteの読者はスキャンするので、接続詞より見出しが効く
```

## 段落展開のパターン

### このアカウントで多用するパターン

| パターン | 使いどころ | 構造 |
|---------|-----------|------|
| 結論先行 | 情報パート全般 | 結論 → 理由 → 具体例 |
| Before/After | 機能紹介 | 今まで → これから → 差分 |
| 問題→解決 | CTA付近 | ペイン提示 → 解決手順 |

```
結論先行の例:
「数式を日本語で指示するだけで作れます。（結論）
 ChatGPT for Excelがセルの内容を読み取り、
 適切な数式を自動生成します。（理由）
 たとえば「B列の合計をC1に」と言うだけ。（具体例）」
```

## 視覚的な段落設計

| 手法 | noteでの実践 |
|------|-------------|
| 空行 | 段落間に必ず空行。見出し前は多めに |
| 見出し | 主題の大きな転換で使用（H2/H3） |
| 図解・画像 | 段落の間に挟んで文字の壁を崩す |
| 掛け合い | テキスト段落が続いた後のブレイク |

**300文字ルール:** テキストが300文字以上続いたら、図解・画像・掛け合いで崩す。
