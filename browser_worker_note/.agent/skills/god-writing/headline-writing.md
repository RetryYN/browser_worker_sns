# 見出し作成（note / X 特化）

読者の目を止め、続きを読ませる見出しの作成技法。
NOT Read（読まない）突破の最重要スキル。

---

## 見出しの重要度

「見出しに費やす時間は全体の50%」（オグルヴィ）

| 要素 | 役割 | このアカウントでの仕事 |
|------|------|---------------------|
| タイトル | noteタイムラインで止まらせる | 「自分に関係ある」と思わせる |
| H2見出し | スキャンで記事の価値を伝える | 見出しだけ読んでも要点がわかる |
| H3見出し | セクション内の案内 | 必要な情報にたどり着かせる |
| X投稿1行目 | タイムラインで止まらせる | 1秒で止める |

---

## タイトルの設計（note記事）

### 文字数: 28〜32文字

noteのタイムラインで全文表示されるギリギリの長さ。

### このアカウントで使うタイプ

| タイプ | 使いどころ | 例 |
|--------|-----------|-----|
| ペイン起点 | ミサキ層の「あるある」から入る | 「Excelの関数、毎回ググってない？」 |
| 事実起点 | ニュース速報 | 「ChatGPT for Excelが来た」 |
| 変化起点 | Before/After | 「VLOOKUP調べる時間、なくなります」 |
| 疑問起点 | 好奇心を刺激 | 「AIがExcelを操作する時代、何が変わる？」 |

### 使わないタイプ

| NGタイプ | 理由 |
|---------|------|
| 煽り型 | 「やばい！神アプデ！」= 驚き屋 |
| 命令型 | 「今すぐ〇〇せよ」= 上から目線 |
| 釣り型 | 「衝撃の結果が…」= 信頼崩壊 |
| 曖昧型 | 「AIの可能性について」= 何の話かわからない |

### タイトル作成プロセス

1. ベネフィットを言語化する（benefit-writing参照）
2. ミサキ層のペインか好奇心を起点にする
3. **10案以上出す**（1つ目で満足しない）
4. 28〜32文字に収める
5. 「自分に関係ある」と思えるか確認

## H2見出しの設計（記事内）

### ルール: 見出しだけスキャンしても記事の要点が伝わる

```
NG（機能名の羅列）:
H2: 数式の自動生成
H2: エラー修正
H2: シナリオ分析
H2: 料金と対応地域
→ マニュアルの目次。ミサキ層は読まない

OK（ベネフィット + 情報）:
H2: 関数を知らなくても数式が作れる
H2: 「なんかエラー出た」をAIが直してくれる
H2: 日本ではまだ使えない。でも今日できることがある
→ 見出しだけで「何が嬉しいか」「制限は何か」がわかる
```

### 見出しに入れるべきもの

| 要素 | 優先度 | 例 |
|------|--------|-----|
| ベネフィット | 最優先 | 「日本語で聞くだけで数式が作れる」 |
| 具体的な数字 | 高 | 「5分 → 30秒に」 |
| 制限・条件 | 高 | 「日本ではまだ使えない」 |
| 読者の言葉 | 中 | 「毎回ググってない？」 |

### 見出しから排除するもの

- 前置き（「ここからは〇〇について」）
- メタコメンタリー（「正直に言うと」）
- 専門用語だけの見出し（「MCP連携」）
- 曖昧な見出し（「その他の機能」「まとめ」）

## X投稿の1行目

### ルール: 1秒で止まる

```
OK:
「Excel関数、毎回ググってない？」
→ ペイン直撃。スクロールが止まる

OK:
「ChatGPT for Excel来た。数式を日本語で作れる。」
→ ニュース + ベネフィット

NG:
「OpenAIがChatGPT for Excelのベータ版を発表しました」
→ プレスリリース。止まらない

NG:
「【速報】ChatGPT for Excel！！」
→ 驚き屋フォーマット
```

---

## 具体性を高める

| 曖昧（NG） | 具体的（OK） |
|-----------|------------|
| 「業務効率化の方法」 | 「VLOOKUP調べる時間を30秒にする方法」 |
| 「AIの新機能」 | 「ChatGPT for Excel — 数式を日本語で作れる」 |
| 「便利なツール紹介」 | 「Excel関数を知らなくても使えるアドイン」 |
| 「まとめ」 | 「今日やれること: ChatGPTにExcelファイルを渡す」 |

---

## 数字の使い方

| 効果的 | 例 |
|--------|-----|
| 手軽さ | 「3ステップで完了」「5分でわかる」 |
| 変化量 | 「5分 → 30秒」「月100分の節約」 |
| 限定 | 「できることは4つ」 |

| 効果が薄い | 理由 |
|-----------|------|
| 「ベンチマーク87%」 | ミサキ層にはスペックシート |
| 「100万トークン対応」 | 技術用語 |
| 「〇〇社が導入」 | B2B訴求。ミサキ層に関係ない |
