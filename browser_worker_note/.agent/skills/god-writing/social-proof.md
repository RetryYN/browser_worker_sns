# 社会的証明（note / X 特化）

「他の人も選んでいる」安心感を読者に伝えるスキル。
NOT Believe（信じない）突破の武器。

---

## このアカウントで使える社会的証明

セールスページではないので、導入企業数や顧客の声はない。
代わりに使える証明を整理する。

| 証明の種類 | 使えるか | このアカウントでの形 |
|-----------|---------|-------------------|
| 数の証明 | △ | 公式発表の数字（「87%の精度」「Plus会員〇万人」） |
| 専門家の証明 | △ | 一次ソース（公式ドキュメント、リリースノート） |
| 類似者の証明 | **○** | **ミサキ層と似た読者の「あるある」共感** |
| 体験の証明 | **○** | **実画面スクショ、やってみた結果** |
| キャラの証明 | **○** | **ChatGPTやClaudeが「自分の機能」として語る** |
| 認証 | × | このアカウントに公的認定はない |

---

## 類似者の証明（最も効果的）

ミサキ層が「自分と同じ人がやっている」と感じる証明。

### 使い方

```
NG: 「AIの専門家が推薦する方法」
→ ミサキ層は専門家に距離を感じる

OK: 「毎朝VLOOKUPでググる人には便利な話です」
→ 「私のことだ」と思える

OK: 「チームの先輩がChatGPTでレポート作ったらびっくりするほど速かった」
→ 日常の風景で証明
```

### 掛け合いでの類似者証明

キャラが読者の代弁者になる:

```
アイコ「毎回VLOOKUP調べるのつらくない？」
→ アイコがミサキ層の「あるある」を代弁
→ 読者: 「わかる…」（共感 = 社会的証明）
```

---

## 体験の証明（最大の武器）

実画面スクショは社会的証明の中で最も強力。

| 証明レベル | 形式 | 信頼度 |
|-----------|------|--------|
| 最強 | 実画面スクショ + 結果 | ★★★★★ |
| 強い | 具体的な操作手順 + 出力例 | ★★★★ |
| 普通 | 「やってみたらこうだった」テキスト | ★★★ |
| 弱い | 「〇〇ができます」説明のみ | ★★ |
| 最弱 | 「すごい！便利！」感想のみ | ★ |

**このアカウントの差別化:** Playwright MCPで撮れる実画面が、驚き屋には出せない証拠になる。

---

## 数字の使い方

公式発表の数字を借りる形で使う。

```
OK: 「財務モデリングベンチマークで87%（OpenAI公式発表）」
→ 出典を明記。数字の権威を借りる

NG: 「87%の精度！」
→ 出典なし。驚き屋と同じ

OK: 「米国・カナダ・オーストラリアの3カ国で提供開始」
→ 具体的な数字は信頼感

NG: 「世界中で話題に！」
→ 曖昧な誇張
```

### ミサキ層に響く数字

| 響く | 響かない |
|------|---------|
| 「30分 → 5分に短縮」 | 「ベンチマーク87%」 |
| 「月5時間の節約」 | 「100万トークン対応」 |
| 「3ステップで完了」 | 「〇〇社が導入」 |

**身体感覚で理解できる数字**を使う。スペックシートの数字は届かない。

---

## キャラによる証明

AI Idol Orchestraのキャラが「自分の機能」として語ること自体が証明になる。

```
ChatGPT「Excel？ 数式も分析も任せて」
→ ChatGPT本人が言っているので「本当にできるんだ」感

Claude「ファイルを渡してもらえれば、今すぐお手伝いできますよ」
→ 実際にClaudeはファイル分析ができる事実
```

**キャラの証言が嘘にならないようにする。** AIの実際の機能に基づいた発言のみ。

## 記事内での配置

```
リード ────── 類似者証明（読者の「あるある」で入る）
本文前半 ──── 数字の証明（公式発表の数字）
本文中盤 ──── 体験の証明（実画面 or 具体的な操作結果）
掛け合い ──── キャラの証明（AIが自分の機能を語る）
CTA ──────── 類似者証明（「同じ悩みの人がやってみた」）
```
