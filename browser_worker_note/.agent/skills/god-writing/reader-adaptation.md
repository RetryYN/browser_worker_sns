# 読者適応（note / X 特化）

読者に合わせて文章を最適化する技法。
god-writing 原則1「読者中心主義」の実践スキル。

---

## このアカウントの読者像（固定）

ミサキ層は固定ペルソナ。毎回ゼロから分析する必要はない。
→ 詳細は `knowledge/account-concept.md` を参照

| 項目 | ミサキ層の特性 |
|------|--------------|
| 知識レベル | ChatGPTは日常使い。API・MCP・エージェントは知らない |
| 関心 | 「本当に使えるの？」「自分でもできるの？」 |
| 立場 | 実務者（事務・営業・マーケ）。意思決定者ではない |
| 時間 | スマホでスキャン。じっくり読まない |
| 感情状態 | 驚き屋に飽きてる。正直な情報が欲しい |
| 離脱ポイント | 「コード書いて」「ターミナル開いて」が出た瞬間 |

## 専門用語の扱い

### ルール: 初出時に翻訳、以降はそのまま

```
初出: 「MCP（AIがツールに接続する仕組み）」
以降: 「MCP」のみでOK

初出: 「アドイン（Excelに追加する拡張機能）」
以降: 「アドイン」のみでOK
```

### 翻訳のレベル感

| 用語 | ミサキ層への翻訳 | NG（技術ブログ調） |
|------|----------------|------------------|
| MCP | AIがツールに接続する仕組み | Model Context Protocol |
| API | AIと外部のソフトをつなぐ窓口 | Application Programming Interface |
| アドイン | Excelに追加する拡張機能 | サードパーティプラグイン |
| ベンチマーク | 性能テストのスコア | 評価メトリクス |
| トークン | AIが処理するテキストの単位 | 入力コンテキストウィンドウ |

### 使わない用語（翻訳しても伝わらない）

- コンパイル、デプロイ、マイグレーション
- パイプライン、CI/CD、コンテナ
- 型定義、リファクタリング、アーキテクチャ

→ これらが必要な記事は「少し先の層」向け。メイン記事では避ける。

## 説明の深さ

### 詳しく説明すべき

| 場面 | 理由 |
|------|------|
| ミサキ層が初めて触れる概念 | 「何それ？」で離脱する |
| 「今日やれること」の手順 | 手順不足で行動できない = NOT Act突破失敗 |
| デメリット・制限の理由 | 理由なしだと「なんで？」で不信感 |

### 簡潔にすべき

| 場面 | 理由 |
|------|------|
| 技術的な仕組みの詳細 | ミサキ層は仕組みより結果が知りたい |
| ベンチマーク数値の詳細 | 「87%」で十分。比較方法の説明は不要 |
| 料金体系の全パターン | Plus以上、でOK。Enterprise向け詳細は余剰 |

```
OK: 「精度87%（OpenAI発表の性能テスト）。前のバージョンより大幅に上がった。」
→ ミサキ層に必要な情報だけ

NG: 「財務モデリングベンチマーク（スプレッドシート上での数式生成・データ分析・シナリオ構築を含む総合評価）において87.3%のスコアを達成。前世代のGPT-5.2では68%であり、19.3ポイントの向上。」
→ 技術ブログ。ミサキ層は読まない
```

## トーンの調整

### このアカウントのトーン（固定）

「AIめっちゃ使ってる先輩が、隣の席で見せてくれる感じ」

| パート | トーン | 例 |
|--------|--------|-----|
| 情報パート | 落ち着いた説明調 | 「ChatGPT for Excelが発表されました。」 |
| 掛け合い | キャラの個性全開 | ChatGPT「Excel？ 任せて。」 |
| CTA | 親しみ + 具体的 | 「まず聞いてみてください。日本語で大丈夫です。」 |

### トーンの境界

```
OK: 「毎回VLOOKUP調べるのつらくないですか？」
→ 先輩が声かけてる感じ

NG: 「VLOOKUPの検索にお悩みではありませんか？」
→ 営業メール

NG: 「VLOOKUP調べるのまじだるくない？」
→ 崩しすぎ。友達ではない
```

## 関心への合わせ方

### エンティティ別の切り口

同じテーマでもエンティティ（読者シナリオ）で入り口が変わる。

| エンティティ | 関心 | 記事の入り口 |
|-------------|------|------------|
| 事務ミサキ | 「Excel関数つらい」 | 「VLOOKUP毎回ググってない？」 |
| 営業ミサキ | 「月次レポート面倒」 | 「レポートの集計、AIにやらせたら？」 |
| マーケミサキ | 「データ分析したい」 | 「Excelのデータ、日本語で質問するだけで分析できる」 |

### 「自分向けだ」と感じさせる

```
OK: 「毎朝VLOOKUPでググる人には便利な話です」
→ 「私のことだ」

NG: 「ビジネスパーソン必見の新機能」
→ 「誰に言ってるの？」

NG: 「データサイエンティストも注目」
→ 「私じゃないな」で離脱
```

## 形式の選択

ミサキ層の好み: ビジュアル + 箇条書き + 短いテキスト

| 形式 | 使いどころ | 優先度 |
|------|-----------|--------|
| 図解・画像 | 抽象概念の可視化、Before/After | 最優先 |
| 箇条書き | 機能リスト、手順 | 高 |
| 表 | 比較（3列以内） | 高 |
| 短いテキスト | 説明パート | 中 |
| 掛け合い | 導入・ブレイク | 中 |
| 長いテキスト | 避ける（300文字ルール） | 低 |

---

## 読む状況の考慮

| 状況 | 対応 |
|------|------|
| スマホでスクロール | 段落2-3行、表3列以内 |
| 通勤中 | スキャンで伝わる見出し設計 |
| 昼休みにnote | 2000-3000文字で完結 |
| Xタイムライン | 1秒で止まるフック |
