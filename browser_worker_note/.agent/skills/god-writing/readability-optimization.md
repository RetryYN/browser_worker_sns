# 読みやすさ最適化（note / X 特化）

読者の認知特性に基づいた文章設計スキル。
NOT Read（読まない）突破の土台。

---

## 前提: 読者はスキャンする

読者の79%はスキャンで拾い読みする。一字一句読む人はいない。
noteのタイムラインではスクロールしながら見出しと画像だけ拾っている。

**設計原則: 「スキャンされても伝わる」構造を作る。**

---

## 認知負荷の軽減

脳の処理能力には限界がある。「読むのに努力がいらない」文章を目指す。

| ルール | note記事 | X投稿 |
|--------|---------|-------|
| 文の長さ | 60文字以内 | 40文字以内 |
| 段落の長さ | 3〜5行 | 2〜3行 |
| 言葉選び | ミサキ層の日常語 | さらに削る |
| 構文 | シンプルに。入れ子にしない | 主語→述語のみ |

### やってはいけないこと

```
NG: 「OpenAIが発表したChatGPT for Excelは、Excelのアドインとして
     動作する自然言語処理ベースのツールで、数式の自動生成や
     エラー修正、シナリオ分析などの機能を提供します。」

OK: 「ChatGPT for Excelが発表されました。
     Excelの中でAIに指示できるアドインです。
     数式の作成やエラー修正をやってくれます。」
```

---

## チャンキング（情報の塊分け）

情報を3〜7個のグループに分割する。

```
NG: 機能を10個箇条書きで並べる
OK: 「できることは4つです」→ 4つに絞って説明
```

**予告する:** 「ポイントは3つ」「4つの機能」と先に数を示す。
読者の脳が受け入れ準備をする。

---

## 視覚的階層

noteエディタで使える視覚要素:

| 要素 | 役割 | 使いどころ |
|------|------|-----------|
| H2見出し | セクションの区切り | 話題が変わるとき |
| H3見出し | 小分類 | H2の中で分ける必要があるとき |
| 太字 | 強調 | 本当に重要な箇所だけ。全体の10-20% |
| 箇条書き | 要点整理 | 3項目以上の並列情報 |
| 表 | 比較・スペック | 2軸以上の情報整理 |
| 引用ブロック | 外部情報 | 公式発表の引用 |

**強調しすぎ注意:** 太字だらけは太字なしと同じ。

---

## 余白（ホワイトスペース）

余白は「無駄」ではなく「デザイン」。

- 段落間に空行を入れる
- 見出しの前に余白を多めに取る
- 300文字以上の連続テキストを作らない
- 図解・画像が視覚的な休息になる

### noteでの余白の作り方

- 段落の区切りで改行を多めに入れる
- 図解を挟んで文字の壁を崩す
- 掛け合いパートが自然な余白になる

---

## 文のリズム

同じ長さの文が続くと単調になる。短→中→やや長→短でリズムをつける。

```
OK:
「ChatGPT for Excelが出ました。
 Excelの中でAIが動きます。
 セルを選んで日本語で指示するだけで、数式を自動生成してくれるアドインです。
 ただし、日本ではまだ使えません。」

NG:
「ChatGPT for Excelが出ました。
 Excelの中でAIが動きます。
 数式を自動生成してくれます。
 ただし日本では使えません。」
→ 全部同じ長さで単調
```

---

## モバイル最適化

noteの読者の大半はスマホで読む。

- 段落は2〜3行で区切る（スマホだと1段落が画面の半分を埋める）
- 表はスマホで横スクロールにならないよう列数を抑える（3列以内推奨）
- 画像は横幅100%で表示される前提で設計する

---

## 注意の維持

読者の注意はすぐ逸れる。維持する方法:

| 手法 | このアカウントでの実践 |
|------|----------------------|
| 区切りで注意をリセット | 図解・掛け合いでブレイク |
| 変化をつける | テキスト → 図解 → 掛け合い → テキスト |
| 質問を投げかける | 「〇〇ってやったことありますか？」 |
| 次への期待 | 「ここまでが機能。で、日本で使えるの？」 |

**300文字ルール:** テキストが300文字以上続いたら、図解・画像・掛け合いで崩す。

---

## チェックリスト

- [ ] スキャンで内容が伝わるか（見出しだけ読む）
- [ ] 一文60文字以内か（X投稿は40文字）
- [ ] 段落は3〜5行か（モバイルで2〜3行）
- [ ] チャンキングされているか（3〜7個のグループ）
- [ ] 太字は全体の10-20%以内か
- [ ] 余白は十分か（300文字以上の壁がないか）
- [ ] 文のリズムに変化があるか
- [ ] モバイルで読みやすいか（表は3列以内）
- [ ] 注意を維持できる構成か（図解・掛け合いで崩し）
