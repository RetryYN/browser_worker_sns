# 先回り対処（note / X 特化）

読者の心の中の「でも…」「本当に…？」を先回りで解消するスキル。
NOT Believe（信じない）突破の武器。

---

## 先回りの原理

- 反論が形成されると心理的抵抗が生まれる
- 先に触れることで「わかっている」感が出る
- 信頼性が上がり、読み進めがスムーズになる
- 読者の疑問を代弁してから解消する

---

## ミサキ層のよくある「でも…」

### AI記事全般への反論

| 反論 | 心理 | 頻度 |
|------|------|------|
| 「また驚き屋の煽りでしょ」 | 広告・誇大表現への警戒 | 高 |
| 「日本で使えないんでしょ」 | 海外先行サービスへの諦め | 高 |
| 「難しそう」 | 技術的ハードルへの不安 | 高 |
| 「有料でしょ」 | コストへの懸念 | 中 |
| 「本当にそんなことできるの？」 | 効果への懐疑 | 中 |
| 「自分には関係ない」 | 自分ごと化の失敗 | 中 |
| 「今じゃなくていい」 | 先延ばし | 低 |

---

## 先回りパターン（このアカウント用）

### パターン1: 「驚き屋でしょ」への先回り

読者がもっとも警戒するポイント。リードで対処する。

```
NG: 先回りしない → 記事を読み始めた時点で疑っている
NG: 「正直に書きます」 → メタ前置きは逆効果

OK: 制限事項をリードに入れる
「〇〇が発表されました。ただし日本ではまだ使えません。」
→ 冒頭でデメリットを出すことで「この人は煽らない」と伝わる
```

### パターン2: 「難しそう」への先回り

ミサキ層の離脱ポイント。専門用語が出た瞬間に対処する。

```
NG: 「MCPプロトコルを介して外部データと連携します」
OK: 「外部のデータと繋がります（金融業界向けの機能です）」

NG: 説明なしで専門用語を使う
OK: 専門用語を使わない。どうしても必要なら括弧で噛み砕く
```

**キャラ掛け合いでの先回り:**
```
Gemini「あ、MCPって何ですか？」
アイコ「外のデータを引っ張ってくる仕組みだよ。今は気にしなくていい」
→ 読者の「わからない」をキャラが代弁して解消
```

### パターン3: 「日本で使えないんでしょ」への先回り

海外先行サービスでは必ず出る反論。

```
OK: リードで制限を出す → 「今すぐ使える方法」を後半で提示
→ 「使えない」で終わらせず「じゃあ今日何ができるか」に繋げる

NG: 制限を記事の後半まで隠す → 「読んで損した」になる
```

### パターン4: 「有料でしょ」への先回り

```
OK: 「無料プランでもできます」を先に書く
OK: 「Plus課金が必要ですが、月〇〇円です」と具体的に

NG: 料金に触れない → 不信感
NG: 「投資対効果を考えれば…」→ セミナー臭い
```

### パターン5: 「本当にできるの？」への先回り

```
OK: 実画面スクショを見せる（最強の証拠）
OK: 「実際にやってみました。結果はこうでした」（体験ベース）
OK: 「精度87%です（OpenAI公式発表）」（一次ソース）

NG: 「すごい」「便利」だけで証拠がない
```

## 記事構成への組み込み方

先回り対処は**読者の「でも…」が浮かぶタイミング**に配置する。

```
リード ────── 「驚き屋でしょ」対処（制限を先出し）
機能説明 ──── 「難しそう」対処（専門用語を噛み砕く）
制限セクション ─ 「日本で使えない」対処（事実 → 代替案へ）
代替案 ────── 「有料でしょ」対処（無料で始められる方法）
代替案 ────── 「本当にできるの？」対処（実画面 or 具体例）
CTA ──────── 「今じゃなくていい」対処（今日やれる小さな一歩）
```

### 掛け合いでの先回り

キャラが読者の「でも…」を代弁する構造:

```
ChatGPT「Excelは私に任せて」
アイコ「日本で使えるの？」          ← 読者の反論を代弁
ChatGPT「……今はアメリカだけ」      ← 正直に答える
アイコ「じゃあ今使えるのは？」      ← 代替案に繋げる
Claude「私ならお手伝いできますよ」  ← 解決策を提示
```

→ 読者が「でも…」と思う前にキャラが言ってくれるので、抵抗が減る
