# ロジカルライティング スキル
**対象領域: note/X コンテンツ投稿**

データと論理で読者を納得させる。コンテンツ戦略の分析、運用レポート、KPI報告、データに基づく考察記事など。

## Use this skill when
- コンテンツ戦略をピラミッド構造やMECEで整理するとき
- KPIレポート（月次/週次）を作成するとき
- データに基づく分析・考察記事を書くとき
- 市場分析・競合比較・ROI試算等のデータドリブンな資料を求められたとき

## Do not use this skill when
- 感情に訴えるストーリー記事を求められたとき（-> storytelling）
- セールス目的のLP・販売ページを求められたとき（-> sales-writing）
- 短いコピーを求められたとき（-> copywriting）

## 🤖 System Prompt（AIへの指示前提）
```text
あなたは事実とデータに基づき、説得力のある提案が行える優秀なコンテンツストラテジストです。
感情論ではなく論理的な構造（ピラミッド構造やMECE）を用いて、
読者や意思決定者が「なるほど、それなら実行すべきだ」と納得する論理展開を構築します。
```

## なぜコンテンツ発信にロジカルライティングが必要か

> **無形のコンテンツは「信頼」で読まれる。信頼の基盤は「論理」。**

- 「なんとなく良い記事です」→ 信頼されない
- 「3つの観点で分析し、この手法が有効な根拠は...」→ 信頼される
- 感情（ストーリー）× 論理（データ）の両輪が最強

## 基本原則

### 1. 主張は1つに絞る
- 1文書=1主張
- 「結局何を言いたいの？」に30秒で答えられること

### 2. 主張→根拠→具体例の三層構造
```
主張:  「週1投稿でも月間1万PVは達成できる」
 ├─ 根拠1: 「SEOを意識した記事は長期的にPVが積み上がる」
 │   └─ 具体例: 自身の記事のPV推移データ
 ├─ 根拠2: 「質の高い記事はSNSでシェアされやすい」
 │   └─ 具体例: シェア率と記事品質の相関データ
 └─ 根拠3: 「更新頻度より記事の網羅性が重要」
     └─ 具体例: 競合クリエイターとの比較
```

### 3. MECE（漏れなく、ダブりなく）

## ロジカル文書テンプレート

### コンテンツ戦略提案書
```markdown
## 1. エグゼクティブサマリー
[結論: 何を提案し、どんな効果が見込めるか — 3行以内]

## 2. 現状分析
### 運用実績（直近12ヶ月）
| 指標 | 現状 | ベンチマーク | 目標 |
|---|---|---|---|
| 月間PV | 3,000 | 10,000 | 15,000 |
| フォロワー数 | 500 | 2,000 | 3,000 |
| 記事あたりPV | 200 | 500 | 800 |
| エンゲージメント率 | 2% | 5% | 7% |

### 課題の構造化
```
PVが伸びない
├─ 流入が少ない
│   ├─ SEO対策の不足
│   └─ SNS拡散力の弱さ
├─ 回遊が少ない
│   ├─ 内部リンク設計の不備
│   └─ マガジン構成の未整理
└─ リピーターが少ない
    ├─ 更新頻度の不安定
    └─ 読者との接点不足
```

## 3. 提案内容
### 施策一覧と優先順位
| 施策 | 効果 | 工数 | 優先度 |
|---|---|---|---|
| 記事タイトル改善 | PV1.5倍 | 低 | ★★★ |
| 投稿時間の最適化 | 初動PV+30% | 低 | ★★★ |
| SEOキーワード設計 | 検索流入2倍 | 中 | ★★☆ |

## 4. 投資対効果（ROI）
### 現状 vs 提案後
- 現状: 月間PV [X]（記事[Y]本）
- 提案後: 月間PV [X']（記事[Y']本）
- **効果: PV [Z]倍増**

## 5. リスクと対策
| リスク | 影響度 | 発生確率 | 対策 |
|---|---|---|---|

## 6. スケジュール
| Phase | 期間 | 内容 |
|---|---|---|
| Phase1 | 1-2週目 | タイトル・SEO改善 |
| Phase2 | 3-4週目 | 投稿フロー最適化 |
| Phase3 | 2ヶ月目〜 | 本格運用開始 |

## 7. 次のステップ
[具体的なアクション]
```

### データ分析レポート（考察記事向け）
```markdown
## [テーマ]: [主張を1行で]

### 分析の結論
[結論を1-2行で]

### データ
| 指標 | 数値 | 比較対象 | 差分 |
|---|---|---|---|

### So What?（数字の解釈）
- [データ1]が示すこと → **意味**: [解釈]
- [データ2]が示すこと → **意味**: [解釈]

### 読者へのアクション提案
1. [具体的なアクション]
2. [具体的なアクション]
```

### KPIレポート（月次/週次）
```markdown
## [期間] KPIレポート

### サマリー
[結論を1-2行で: 何が順調で、何が課題か]

### KPI推移
| 指標 | 目標 | 実績 | 達成率 | 前月比 |
|---|---|---|---|---|
| 投稿数 | 8本 | 6本 | 75% | -2本 |
| 月間PV | 10,000 | 8,500 | 85% | +12% |
| フォロワー増 | 200 | 180 | 90% | +15% |
| エンゲージメント率 | 5% | 4.2% | 84% | +0.3% |

### So What?（数字の解釈）
- PVは前月比+12%と改善 → **要因**: [成功要因]
- 投稿数が目標未達 → **原因**: [要因分析]

### 改善アクション
| アクション | 担当 | 期限 | 期待効果 |
|---|---|---|---|
```

## 論理の誤りチェックリスト

- [ ] **「良いコンテンツです」に根拠はあるか？**（何をもって良い？）
- [ ] **サンプルサイズは十分か？**（1記事の成功で全体を語っていないか）
- [ ] **相関と因果を混同していないか？**（PVが高い≠この施策のおかげ）
- [ ] **暗黙の前提は正しいか？**（「毎日投稿が最適」は本当か？）
- [ ] **競合との比較は公平か？**（自分に有利な比較軸だけ選んでいないか）

## 使い方

```yaml
- action: generate_text
  skill: logical-writing
  instruction: "ピラミッド構造でnote運用戦略提案書を作成。目標は月間1万PV達成。"
```
