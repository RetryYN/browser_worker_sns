```
1. select_option "月" → 月を選択

5. select_option "分" → 分を選択

8. click "予約設定する" → 予約完了
```

3. select_option "月" / "日" / "年" / "時" / "分" → 各combobox選択
4. click "確認する" → モーダルに戻る
5. type（slowly: true）→ 本文入力
6. click "画像や動画を追加" → ファイルチューザー検出
7. file_upload → 画像パス指定
8. [人間承認]
9. click "予約設定"

### 共通の注意点

- **日本語入力**: `browser_type` に `slowly: true` 必須
- **画像添付**: click → file_upload の2ステップ。ファイルチューザーは自動検出される
- **成功検出**: alert 要素。即時="ポストを送信しました。"、予約="ポストの送信日時: ..."
- **ref値は毎回変わる**: snapshot で都度取得。ハードコードしない

### 実証済みフロー — プロフィール更新

```
1. navigate /UaW6wnKW8c87193 → プロフィールページ
2. click "プロフィールを編集" → 編集モーダル表示
3. click "バナー画像を追加" → ファイルチューザー
4. file_upload → ヘッダー画像パス
5. click "適用" → クロップ確定、モーダルに戻る

7. file_upload → アイコン画像パス
8. click "適用" → クロップ確定、モーダルに戻る
9. click 名前テキストボックス → Ctrl+A → type で上書き
10. 自己紹介は evaluate で直接値セット（slowly: true だとタイムアウト）
11. click "保存" → プロフィールページに戻る
```

### 注意点

- **自己紹介(textarea)**: `browser_type` の `slowly: true` はタイムアウトする。`browser_evaluate` で `nativeInputValueSetter` + `input` イベントで直接値セットする
- **画像クロップ**: アップロード後に「メディアを編集」ダイアログ（スライダー付き）が表示される。デフォルト位置でよければ「適用」を押すだけ
- **名前の上限**: 50文字。カウンター表示あり
- **自己紹介の上限**: 160文字。カウンター表示あり

## 注意事項

- **ログインフロー**: 2段階（メール入力画面 → パスワード入力画面）。各画面でのref値はセッションごとに変わる
- **2要素認証ダイアログ**: ログイン直後に「アカウントのセキュリティを強化する」が出る場合あり。「閉じる」で回避
- **初回ロード**: SPAのため5秒待機推奨
- **ポスト文字数制限**: 140文字（プレミアムなら拡張可能）
- **beforeunloadなし**: noteと違い、X側では離脱時のダイアログなし
- **予約投稿**: プレミアム不要。comboboxで日時指定
- **ブラウザセッション**: context断絶後もブラウザは維持される。再ログイン不要の場合あり
