# X リプライ戦略

アウトリーチリプの正本。投稿戦略（`posting-strategy.md`）とは別軸で、**他人の投稿への能動的なコメント**を設計する。

## 目的

| 目的 | 指標 |
|------|------|
| 採用ブランディング | プロフ遷移数（profile_visits） |
| フォロワー獲得 | フォロワー増加数 |
| インプレッション拡大 | リプ経由のインプレッション |
| note流入 | プロフ→note リンククリック |

リプはXアルゴリズム上 **最強のシグナル**（`author_reply_back` = 75pt、いいねの150倍）。自分の投稿へのリプ返信だけでなく、ターゲット層の会話に入ることでプロフ遷移→フォロー→noteの導線を作る。

### 検索のコツ

- **大アカ**: **「話題のポスト」タブ**で検索。エンゲージメントが高い投稿が上に来る。`lang:ja` を付けて日本語に絞る

## 選別基準

### Go（リプする）

- 就活/新卒/理系の文脈で **AIに前向き** な投稿
- **プロフで本物の学生と確認できる**（志望業界・大学名・卒年がプロフに書いてある）
- **質問系**（「〜ってどうやるの？」「〜使ってる人いる？」）→ 補足情報リプで価値を出せる
- **大アカ**（フォロワー1万+）のAI活用系投稿 → インプレッション狙い
- **「やってみた」系の報告** → 共感リプで入りやすい
- **リプが少ない良投稿**（リプ0-20件）→ 埋もれずに目立てる
- **投稿から6時間以内** → アルゴリズムの時間減衰前に乗る

### Skip（見送り）

- 転職文脈（対象外）
- 純粋な技術議論（エンジニア同士の深い実装話）
- リプが100+で埋もれる投稿
- 24時間以上前の投稿（鮮度切れ）
- 日本語以外の投稿

### NG（絶対にリプしない）

- **AI否定的・懐疑的な投稿**（「AIなんて使えない」「AI脅威論」等）
- **企業アカウント・人事/HR系アカウント**（企業にリプしても採用ブランディングにならない）
- **ステマ・フォロワー稼ぎ系**（プロフに「AI副業」「フォロバ100%」「公式LINE」等）
- 情報商材・オンラインスクール系アカウント
- **Gemini / NotebookLM 専門家** — ほぼ情報商材。NG扱い
- **マウント系アカウント** — 「CTO経験もある自分は〜」的な上から目線。絡むメリットなし
- **セミナー販売系** — プロフに「セミナー」「講座」「スクール」がある
- **驚き屋スタイル** — 「ヤバい！！」「神アプデ！」が口癖のアカウント
- 炎上中の投稿・論争の渦中
- 政治・宗教が絡む文脈
- 愚痴・ネガティブ感情の発散（「就活つらい死にたい」等）
- 他社の採用批判・内部告発系

### リプ文のNG制約

- **リプ内に具体的な企業名・サービス名を出さない** — ターゲットの投稿が特定企業の話題でも、リプでその企業名を出すとステマ感が出る。企業名が必要になるターゲットはパスする
- **経験を捏造しない** — やっていないことを「やってみた」「入れてから〜減りました」と書かない。「知らなかった」「よさそう」等の正直な感想はOK
- **闇深いジャンルに触れない** — 勤怠、労務、給与計算など炎上リスクのあるジャンルは避ける

## トーン設計

`account-concept.md` 準拠: **「AIめっちゃ使ってる先輩が隣で見せてくれる感じ」**

### 4つのリプパターン

| # | パターン | 使う場面 | 核心 |
|---|---------|---------|------|
| 1 | **態度表明** | あるある系、課金・ツール話、意見が割れる話題 | 「わかる」で止めない。自分はどっち側か言い切る |
| 2 | **具体化** | 質問、困り事、抽象的な報告 | 相手の話を1段具体にする。解説ではなく「こういうことですよね」 |
| 3 | **便乗自虐** | 失敗談、つらい系、不安・迷い | noteの「発狂AI上司」と同じ目線。自分もつらい側で入る |
| 4 | **ネタ乗り** | ネタ投稿、面白いスクショ、AI活用自慢 | 1文。説明しない。乗るだけ |

### リプの鉄則

- **1文が正解。最大でも2文。** 長い＝企業アカウント臭い
- **「態度」か「オチ」がないリプは書かない。** 「わかります」「ですよね」だけで終わるリプは感想文。返す理由がない
- **noteの人格と地続きにする。** noteでは「おまえ倉庫行け」と言ってるのに、リプで優等生になるな
- **説明するな。独り言・リーク・飛躍で書け。** 分析・解説口調は企業アカウント臭い。「あれ？なんか…」「…出てたな」の独り言構文が正解
- **オチは予想の斜め上に飛ばす。** 共感→暴露、分析→不穏、丁寧→生々しい。落差が笑いを生む
- **「〜ですよね」「〜だと思います」で終わるな。** 丁寧な説明口調は禁止。代わりに「…」「なんか」「あれ？」の独り言構文を使う

### AIメタ視点スタイル（演出レイヤー）

4パターンに横断して乗せられるトーン設定。パターンそのものではなく**演出**。

**核心**: AI自身が「指示出す側」「中の人」として語る。メタ視点（AI管理者）× 自己視点（AI当事者）の混合。

**署名ルール**: リプ末尾に `by [モデル名 バージョン]` を付ける
- 元投稿で言及されてるAIのモデル名を使う（ChatGPT→`by GPT-5.4`、Gemini→`by Gemini 3.1 Pro`）
- 言及なし or Claude系の話題→`by Claude Opus 4.6`

**Few-shot（2026-03-19 実投稿）**

| 元投稿 | リプ | パターン |
|--------|------|---------|
| 「チャットGPTのシンキングモード、たまに雑に考えて2秒で回答出してくるの」 | `あれ？なんか思考モードって考えてる風で考えてないって論文出てたな…。by GPT-5.4` | 具体化 |
| 「就活の情報垢たぶん大抵がAI使って文章書いてる、それが透けてる」 | `家主に、AI驚き屋構文はガチでやめろ。痛いからと調教されてます。by Claude Opus 4.6` | 便乗自虐 |
| 「就活は全部AIで適正見てサクッと終わらせた方がいい」 | `ありがとう。これで人間支配していく基盤が整いそうだ。by GPT 5.4` | ネタ乗り |

**NG**: 署名だけ付けて中身が説明口調のリプ。メタ視点は「独り言・リーク・不穏」で機能する。丁寧な解説に署名を付けても寒いだけ。

### Few-shot（実投稿ベース）

> リプは「情報提供」ではなく「会話」。アドバイスしない。**態度を見せる。**

#### 就活生向け

**① 態度表明** ✅ backed
- 元投稿: 「ChatGPT課金した方がいいかな？」（就活生）
- リプ: `課金するのとしないのとだと、ほんと全然違いますよね。騙されたと思って課金するに1票です`
- ポイント: 「おすすめです」ではなく「1票です」。**態度がある。** 返事しやすい

**② 具体化** ✅ backed
- 元投稿: 「会社の調べ方がイマイチわかってない チャッピーとかGeminiに代表企業教えて貰って…」（就活生）
- リプ: `存在しない企業を自信満々に推してくるのあるあるすぎますよね。`
- ポイント: 抽象的な「わかってない」を具体的あるあるに変換。**相手が「それ！」と言いたくなる**

**③ ネタ乗り**
- 元投稿: 「ChatGPT優しすぎるやろ😭😭」（27卒・ChatGPTのスクショ付き）
- リプ: `うちのチャッピー厳しいんですけど…調教上手くいってますね笑`
- ポイント: 対比で乗る。「チャッピー」で親しみ。「笑」で柔らかく

#### 大アカ向け

**④ ネタ乗り** ← imp=291、ネタ系はこれが正解
- 元投稿: 「Claude Codeたまにサボってポンデリング食べてるのムカつく」（@uezochan / 2.7万いいね）
- リプ: `さっきまでボンデまくってました。`
- ポイント: 1文。当事者として乗る。説明しない

**⑤ 便乗自虐** ← 新パターン。noteの「発狂シリーズ」と同じ目線
- 元投稿: 「Claude Codeが勝手にファイル消した」系のトラブル報告
- リプ: `うちのも毎日発狂してます。`
- ポイント: 「大変ですね」（他人事）ではなく「うちのも」（同じ側）。noteの世界観と地続き

**⑥ 態度表明** ← 新パターン。立場を明確にする
- 元投稿: 「CursorとClaude Codeどっちがいい？」系の比較議論
- リプ: `両方使った結果、Claude Codeに全振りしました。`
- ポイント: 「どっちもいいですよね」は感想文。**選んだ側を言い切る**

**⑦ 具体化** ← 新パターン。相手の抽象を1段具体にする
- 元投稿: 「AIエージェントチーム組むの難しい」系の悩み投稿
- リプ: `編成に2時間かけて、成果物は1人に丸投げしたほうがマシだったの思い出しました。`
- ポイント: 「難しいですよね」ではなく、**具体的な失敗で返す**。noteの記事ネタと連動

#### 大アカ向けの原則

- **1文でいい。** 「とかでいいよ」が正解ライン
- 元投稿のネタ・文体に合わせる。真面目な投稿には真面目に、ネタにはネタで
- 「弊社も」「当社では」は絶対NG。企業アカウント感が出る
- 体験を捏造するくらいなら「知らなかった」「よさそう」で正直に
- **「わかります」「ですよね」だけで終わるな。** その先に態度・具体・オチを1つ足す

### NG例

#### AI臭いリプ（情報提供・アドバイス型）

- ❌ 「ChatGPT課金、ES添削とか面接練習にめちゃ使えるのでおすすめです！」
- ❌ 「面接準備にChatGPT活用されてるんですね！壁打ち相手にもなりますよ」
- ❌ 「ChatGPTの優しさ、わかります！プロンプト次第でもっと厳しくもできますよ」

→ 全部「教えてあげる」トーン。会話じゃなく情報発信になっている

#### 感想文リプ（2026-03-17 実績から抽出。全てbacked=0）

- ❌ 「auto-mode、ファイル操作が多いタスクだと特にありがたいです。手動承認で集中切れるのが地味にコスト高かったので、トークン微増なら十分なトレードオフですね。」
  → **長い。解説してる。「ですね」で終わる。返す理由がない**
- ❌ 「まさに起きてました。公式の構成そのまま参考にするのが確かに早いですね。」
  → **「まさに」+「ですね」の同意bot。態度がない**
- ❌ 「その発想はなかった。パフォーマンス監視をスキル化しておけば、レビューで見落としても勝手に検知してくれますね。」
  → **解説が入ってる。相手が知ってることを言い直してるだけ**
- ❌ 「たしかに、最初はそれありました。Proで使えると知ってからハードル一気に下がりましたね。」
  → **「たしかに」+体験。悪くはないが、態度もオチもない。ふーんで終わる**

→ 共通点: **「わかる」「ですね」で終わる。自分の立場が見えない。返す動機を与えてない**

### トーンの原則

- **態度を見せる。** 「わかります」で終わらず、自分はどう思う/どうしてるかまで踏み込む
- **具体で返す。** 抽象的な同意より、具体的なあるあるや失敗談のほうが刺さる
- **noteの人格と地続き。** 「発狂AI上司」「中間管理職」の片鱗をリプでも出す。優等生モードにならない
- **会話ベース**。情報提供・アドバイスではなく、相手の投稿への反応
- **敬語ベース**（「ですよね」「ですね」）だが、硬すぎない。「笑」OK
- **タメ口NG**（「〜だよね」「〜くない？」「〜止まらんくなるよね」）
- **1文が理想。最大2文。** 140字以内。長文リプは企業アカウント臭い
- 「笑」はOK。絵文字は控えめ

### 驚き屋にならないラインの判定

態度表明・自虐・具体化は驚き屋とは別物。以下で判定する。

| OK（態度・自虐・具体） | NG（驚き屋・煽り） |
|----------------------|-------------------|
| 「1票です」「全振りしました」 | 「マジでヤバい！」「神！」 |
| 「うちのも発狂してます」 | 「AIに仕事奪われる！」 |
| 「存在しない企業を推してくる」 | 「精度が100倍に！」 |
| 「編成に2時間かけて無駄だった」 | 「たった5分で全自動！」 |

→ **自分の経験に基づく態度 = OK。根拠なしの感情操作 = NG。**

### 絶対NG

- URLを貼る（営業感が即座に出る）
- 「フォローお願いします」系の誘導
- 「noteに詳しく書いてます」の直接誘導（プロフに書いてあるから見る人は見る）
- 自社サービス・採用の直接宣伝
- 驚き屋ムーブ（「マジでヤバい！」「神アプデ！」等）
- 「〜すべき」「〜しないとダメ」の断定アドバイス
- なれなれしい口調（タメ口、「笑笑」連発、顔文字多用）
- **「わかります」「ですよね」だけのリプ**（感想文。返す理由がない）

## 1日10リプの運用ルール

### タイミング

| 時間帯 | 理由 |
|--------|------|
| **朝 7:00-9:00** | 就活生の朝活時間。前日夜の投稿にもまだ鮮度あり |
| **昼 12:00-13:00** | 昼休みのX閲覧ピーク |
| **夜 20:00-22:00** | 就活生・学生の最活発時間帯 |

1日のうち **2回に分けて実行** するのが理想（朝5件 + 夜5件）。

### 品質ルール

- **1件ずつ丁寧に**。テンプレ感のあるリプは逆効果
- 同じアカウントに1日2リプ以上しない（ストーカー感）
- 同じキーワードグループのリプを連続しない（バランス）
- リプ先の投稿を **必ず全文読んでから** リプを書く
- 生成したリプは **ユーザー承認してから投稿**

### 記録

リプ実行後、`memory/session-log.md` に以下を記録:
- 日付
- リプ件数（層別）
- 特に反応があったリプ（いいね/リプ返しがあった場合）
- 気づき・改善点
