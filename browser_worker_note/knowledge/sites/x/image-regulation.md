# X 画像レギュレーション

用途はサムネイル、図解（note転用）、Xグリッド投稿、クイズの4カテゴリ。
図解は note で生成したものを転用する（note が正本）。

キャラクター詳細は `knowledge/characters/team-orchestra.md` を参照。

## サムネイル

| 項目 | 仕様 |
|------|------|
| 用途 | note記事シェア時のOGP / 単発ポストの添付画像 |
| サイズ | note サムネイル（1536×1024px）をそのまま使用。比率がほぼ同じため問題なし |
| 容量 | 5MB以下 |
| スタイル | 粗めのピクセルアート、大きなピクセルブロック。アンチエイリアスなし |
| 構図 | 中央寄せ、重要要素は中央60%に収める（自動トリミング対策） |
| 必須要素 | アイドルキャラ + ビジネスマンの共存 |
| カラー | アカウントパレット準拠 |
| テキスト | 画像内にテキストなし |
| NG | 写真風、リアル調、AI単体のみ、小さすぎるディテール |

## 図解（note から転用）

| 項目 | 仕様 |
|------|------|
| 用途 | ポスト内の概念説明 |
| サイズ | 1024×1024px（note で生成した正方形をそのまま使用） |
| 容量 | 5MB以下 |
| カラー | タイムラインで目立つ彩度高め |
| テキスト | 短いラベルのみOK（日本語可、最大5語） |
| NG | 情報過多、スマホで潰れる細部 |
| 備考 | note の図解をそのまま転用。X専用に作り直さない |

## Xグリッド投稿（4枚で1ストーリー）

> リファレンス: アイコ（`knowledge/characters/aiko/ref_idol.jpg`）のトーンに準拠

| 項目 | 仕様 |
|------|------|
| 用途 | X投稿専用。4枚の独立画像を2×2グリッドで投稿し、ストーリーを伝える |
| サイズ | 1536×1024px（API出力そのまま、3:2横長） |
| 容量 | 各5MB以下 |
| スタイル | ピクセルアート、やや精密。アンチエイリアスなし |
| 構図 | 各画像が独立した1シーン。1画像1アクション。小さくても読める構図 |
| 必須要素 | アイドルキャラ + ビジネスマンが共演。キャラデザインは4枚で統一 |
| キャラ選択 | 話題に合ったメンバーを選択（デフォルトはアイコ） |
| ストーリー | 起承転結の4ビート（左上→右上→左下→右下の読み順） |
| カラー | 紫〜ピンクのグラデーション背景 |
| テキスト | 各画像下部に日本語ラベル（高コントラストのバナー内、2-4文字推奨） |
| 生成方式 | **独立4回生成**（Method C）。1枚の分割ではなく、各シーンを個別にAPI生成 |
| NG | 写真風、リアル調、1枚の画像を分割する方式（ギャップで破綻する） |

## クイズ画像（8タイプ）

> リファレンス: アイコ（`knowledge/characters/aiko/ref_idol.jpg`）のトーンに準拠

### 共通仕様

| 項目 | 仕様 |
|------|------|
| 用途 | X投稿専用。エンゲージメント獲得（リプライ誘導）のためのクイズカード |
| サイズ | 1024×1024px（正方形） |
| 容量 | 5MB以下 |
| スタイル | ピクセルアート、やや精密。アンチエイリアスなし |
| カラー | 紫〜ピンクのグラデーション背景。明るく楽しいトーン |
| 必須要素 | アイドルキャラ（1体、小さめ、プレゼンター役）+ クイズ内容 + CTA（行動喚起バナー） |
| キャラ選択 | アイコ or 話題に合ったメンバー |
| NG | 写真風、リアル調、オフィス/アイソメトリックシーン、キャラ複数体 |

### 8タイプ一覧

| タイプ | CLI名 | レイアウト | CTA例 |
|--------|-------|-----------|-------|
| 選択式 | `quiz-choice` | 問題文 + A/B/C/D色分け2×2グリッド | コメントで回答! |
| ○×クイズ | `quiz-ox` | 命題文 + ○×大ボタン左右配置 | ○か×でリプライ! |
| 穴埋め式 | `quiz-fill` | 文中に光る空欄ボックス | 答えをリプライ! |
| ランキングあて | `quiz-ranking` | 表彰台/番号リスト + ???隠し枠 | 1位を予想してリプライ! |
| 間違い探し | `quiz-spot` | 左右分割の類似シーン（3箇所の違い） | 違いを3つ見つけてリプライ! |
| 設問式 | `quiz-open` | 装飾フレーム内に質問文 | あなたの答えをリプライ! |
| 数値あて | `quiz-number` | 光る???ボックス + ゲージバー | 数字を予想してリプライ! |
| Before/After | `quiz-ba` | 左右Before/After対比 + →矢印 | 正しいのはどっち？リプライ! |

### クイズ固有の設計原則

- **正方形（1:1）**: Xタイムラインで目立つ。クロップ不要で全要素が必ず表示される
- **CTA必須**: 画像下部に行動喚起バナーを配置し、リプライを誘導する
- **`--quiz-data`**: 選択式・ランキングでは選択肢/項目をCLI引数で渡す
- **エンゲージメント優先**: 正解より「つい答えたくなる」デザインを重視

## X 固有の画像仕様

| 枚数 | 表示比率 | 推奨サイズ |
|------|---------|-----------|
| 1枚 | 16:9（横長） | 1200×675px |
| 1枚 | 3:4（縦長、目立つ） | 1200×1600px |
| 1枚 | 1:1（正方形） | 1200×1200px |
| 2枚 | 各 3:4 | 900×1200px |
| 3枚 | 1枚目 3:4 + 2-3枚目 16:9 | 900×1200 + 1200×675px |
| 4枚 | 全て 16:9 | 1200×675px |

## API生成仕様

| 項目 | Xグリッド | クイズ(全8種) |
|------|----------|-------------|
| API生成サイズ | 1536×1024（横長） | 1024×1024（正方形） |
| 後処理 | なし | なし |
| API呼び出し回数 | 4回（パネルごと） | 1回 |

- モデル: gpt-image-1.5（`images.edit()` API）
- API: `OPENAI_API_KEY` をプロジェクト `.env` から取得（ファイルに書かない）
- 品質: `quality="high"`（テキスト精度・ディテール向上）
- リファレンス画像: `knowledge/characters/aiko/ref_idol.jpg` をfew-shotとして添付（デフォルト）
- 形式: JPG（API から PNG 取得後、Pillow で JPG 変換して保存。軽量化のため）
- プロンプト構造: `[Reference Image]ラベル` → タイプ別ルール → ユーザーの依頼

## 保存ルール

- 保存先: `knowledge/data/images/`（note と同じフォルダ）
- 命名:
  - Xグリッド: `x_grid_panel-{1-4}_{topic}_{YYYY-MM-DD}.jpg`
  - クイズ: `x_{quiz-type}_{topic}_{YYYY-MM-DD}.jpg`
  - 例: `x_grid_panel-1_meeting-minutes_2026-03-11.jpg`
  - 例: `x_quiz-choice_ai-task_2026-03-11.jpg`
- 1ポストあたり上限: 画像4枚まで（X 仕様）
- 形式: JPG
