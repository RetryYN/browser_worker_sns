# note サムネイル（見出し画像）ガイド

note公式仕様・外部リサーチに基づくサムネイルデザインの知見集。

## 公式仕様

| 項目 | 値 |
|------|-----|
| 推奨サイズ | **1280×670px** |
| アスペクト比 | **1.91:1**（≒16:9） |
| 最大容量 | 10MB |
| 形式 | JPEG / PNG |

## 表示場所ごとの挙動

| 表示場所 | 挙動 |
|---------|------|
| タイムライン（グリッド） | 1280×670 にクロップ表示。上下が切れやすい |
| 記事ページ（PC） | 画面幅いっぱいに表示 |
| 記事ページ（スマホ） | 画面幅に縮小。上下クロップが発生する場合あり |
| SNSシェア（OGP） | 見出し画像がそのまま OGP として使用される |

**重要**: noteはタイトルを画像の上にオーバーレイしない。画像とタイトルは別要素として表示される。

## セーフゾーン

- 重要な要素（テキスト・顔・ロゴ）は**画像中央に配置**
- 上下左右の端から **10〜15%** を余白として確保
- スマホ表示では上下がクロップされるため、天地方向の余白が特に重要
- 3:2画像を1.91:1にクロップすると、上下合計で約 **130px**（1024px高の場合）が切れる

## テキスト・イン・イメージ

| 項目 | 推奨 |
|------|------|
| 文字数 | 短いキャッチフレーズ（10文字以内が理想） |
| フォント | 太字ゴシック体。小さすぎない |
| 配置 | 中央寄せ。端は避ける |
| コントラスト | 背景と文字色のコントラストを強く |
| 補助手段 | シャドウ・縁取り・半透明帯で視認性確保 |

**3秒ルール**: タイムラインで一瞬で内容が伝わるシンプルさが最重要。

## デザインパターン（一般的な note 記事）

| パターン | 特徴 | 向いている記事 |
|---------|------|---------------|
| 単色背景＋大きめフォント | シンプル・高インパクト | ビジネス・ノウハウ |
| 写真背景＋テキストオーバーレイ | 人物で視線誘導 | エッセイ・ライフスタイル |
| イラスト・図解系 | テーマの直感的理解 | テック・解説 |
| ダークカラー＋コード風 | エンジニア向け | 技術記事 |

## 配色ルール

- **3色以内**（メイン・サブ・アクセント）
- 背景と文字のコントラストを明確に
- ブランドカラーを統一するとフォロワーの記憶に残る

## 当プロジェクトへの適用

### アスペクト比の対応

API出力は **1536×1024 (3:2)** で固定。note推奨の **1.91:1** に合わせるため、後処理でクロップする。

```
元画像:   1536 × 1024 (3:2)
  ↓ 上下クロップ（天地均等）
クロップ:  1536 × 804  (1.91:1)
  ↓ リサイズ
最終出力:  1280 × 670  (1.91:1)
```

- クロップ量: 上下合計 220px（上110px・下110px）
- キャラの足元や頭上の余白がクロップされるため、プロンプトでセーフゾーンを意識させる

### 当プロジェクトのスタイルとの整合

ピクセルアートスタイルは note の一般的なパターン（写真・Canvaテンプレ）とは異なるが、差別化要素として有効。ただし以下に注意:

- テキストはスタイル固有のもの（チョーク文字・ゲームUI等）に限定（レギュレーション既定）
- セーフゾーンを考慮したキャラ配置（中央寄せ）
- ピクセルアートは小さく縮小されると潰れやすいため、太いピクセルブロックを維持

## 参考 og:image URL パターン

```
https://assets.st-note.com/production/uploads/images/{image_id}/{hash}.{ext}
```

- CDN: `assets.st-note.com`
- 形式: PNG が多数、JPG も利用可
- OGP にはサイズパラメータなし（原寸参照）
