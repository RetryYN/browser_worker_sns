# note 画像レギュレーション

用途はサムネイル（アイキャッチ）、図解、4コマ漫画の3カテゴリ。
図解・4コマ漫画は X でも転用可（note が正本）。

キャラクター詳細は `knowledge/characters/team-orchestra.md` を参照。
サムネイルのnote表示仕様・デザイン知見は `knowledge/sites/note/thumbnail-guide.md` を参照。

## サムネイル

> リファレンス: アイコ（`knowledge/characters/aiko/ref_idol.jpg`）のスタイルに準拠

| 項目 | 仕様 |
|------|------|
| 用途 | 記事の見出し画像（アイキャッチ） |
| サイズ | **1280×670px**（API出力 1536×1024 → クロップ＆リサイズ。note推奨 1.91:1） |
| 容量 | 10MB以下 |
| スタイル | 粗めのピクセルアート、大きなピクセルブロック。アンチエイリアスなし |
| 必須要素 | アイドルキャラが登場（スタイルにより構図が異なる） |
| テキスト | 画像内テキストはチョーク文字・ラベル等スタイル固有のものに限る（noteはタイトルを画像にオーバーレイしない。別要素で表示） |
| セーフゾーン | 重要要素は画像中央に配置。上下110pxはクロップされるため天地に余白を確保 |
| NG | 写真風、リアル調、文字だらけ、ダークネイビー一色背景、冷たい色調、**キャラの幼児化（幼稚園児・園児体型に改変すること）** |

## 図解

> リファレンス: アイコ（`knowledge/characters/aiko/ref_idol.jpg`）のトーンに準拠

| 項目 | 仕様 |
|------|------|
| 用途 | 記事内の概念・手順の視覚化（X ポストでも転用） |
| サイズ | 1024×1024px（正方形） |
| 容量 | 5MB以下（X転用を考慮） |
| スタイル | ピクセルアート、やや精密。アンチエイリアスなし |
| 構図 | **説明対象（概念図・フロー・アイコンなど）が中央**。アイコ（ピンクツインテール、紫アイドル衣装、金星ブローチ）は端（左下 or 右下）に小さめに配置し、指差し・紹介ポーズで解説役を担う。主役はあくまで図解内容 |
| 必須要素 | アイコ + 説明対象（概念図・フロー・アイコンなど） |
| カラー | 紫〜ピンクのグラデーション背景 or 薄色背景。キャラのピンク・パープル・ゴールドが映える配色。視認性優先 |
| テキスト | 日本語ラベルOK（高コントラストの枠付きバナー内に配置）。矢印は最小限。吹き出しは使わない |
| NG | 情報過多、3概念以上の詰め込み、キャラなしの図だけ、写真風、オフィス/アイソメトリックシーン |

### 図解レイアウト: ボード型（`--layout board`）

複雑な図解（フロー図・比較表・手順リスト等）を描く場合に使用するオプションレイアウト。

| 項目 | 仕様 |
|------|------|
| 構図 | 画面の60-70%を占める黒板/ホワイトボードを配置。図解内容はボード面上に描画（チョーク風/マーカー風） |
| キャラ配置 | ボードの完全に外側（左端 or 右端の下部）に立ち、指差しポーズで解説。**ボード上のテキスト・図と絶対に重ならない**。キャラとボード端の間にクリアな隙間を確保 |
| 適用場面 | フロー図、比較表、手順リスト、段階図など、明確な描画面が必要な図解 |
| 利点 | 描画面が明確になりピクセルアートでも情報が整理される。ボードフレームが自然な区切りになる |

## 4コマ漫画

> リファレンス: アイコ（`knowledge/characters/aiko/ref_idol.jpg`）のトーンに準拠

| 項目 | 仕様 |
|------|------|
| 用途 | 記事内の挿絵・X投稿用のストーリー型ビジュアル |
| サイズ | 1024×1536px（縦長 2:3） |
| 容量 | 5MB以下 |
| スタイル | ピクセルアート、やや精密。アンチエイリアスなし |
| 構図 | **4つの等分パネルを縦に積む**（上→下の読み順）。パネル間は黒い細枠で区切る |
| 必須要素 | アイドルキャラが全4パネルに一貫したデザインで登場。ビジネスマンが共演 |
| キャラ選択 | 話題に合ったメンバーを選択（デフォルトはアイコ）。4パネル内でキャラデザインを統一 |
| ストーリー | 起承転結の4ビート。最終パネルにオチ（ユーモア推奨） |
| カラー | 紫〜ピンクのグラデーション背景。パネルごとに微妙に変化OK |
| テキスト | 各パネル下部に日本語ラベル（高コントラストのバナー内、2-4文字推奨）。吹き出しは使わない |
| NG | 写真風、リアル調、パネル数が4以外、キャラデザインの不一致、横読み構図 |

## API生成仕様

| 項目 | サムネイル | 図解 | 4コマ漫画 |
|------|-----------|------|----------|
| API生成サイズ | 1536×1024（横長） | 1024×1024（正方形） | 1024×1536（縦長） |
| 後処理 | クロップ→1280×670（1.91:1）にリサイズ | なし | なし |
| API呼び出し回数 | 1回 | 1回 | 1回 |

- モデル: gpt-image-1.5（`images.edit()` API）
- API: `OPENAI_API_KEY` をプロジェクト `.env` から取得（ファイルに書かない）
- 品質: `quality="high"`（テキスト精度・ディテール向上）
- リファレンス画像: スタイルに応じて `knowledge/characters/*/ref_idol.jpg` を自動選択（マルチリファレンス対応）
  - 単体キャラ向けスタイル: アイコの ref のみ添付
  - 全員集合スタイル（idol-stage, game-start）: 5キャラ分の ref を添付
  - 対決スタイル（vs-battle）: 対決する2キャラの ref を添付
  - `--chars` CLI オプションでデフォルトを上書き可能
- 形式: JPG（API から PNG 取得後、Pillow で JPG 変換して保存。軽量化のため）
- プロンプトのベース: `knowledge/account-concept.md` のカラーパレット・デザイン原則に従う
- プロンプト構造: `[Reference Image]ラベル` → タイプ別ルール → ユーザーの依頼
- **重要**: 各タイプのルールを混在させない（各タイプ専用プロンプトのみ注入）

## 保存ルール

- 保存先: `knowledge/data/images/`
- 命名: `note_{type}_{topic}_{YYYY-MM-DD}.jpg`
  - 例: `note_thumbnail_ai-agent_2026-03-11.jpg`
  - 例: `note_diagram_mcp-flow_2026-03-11.jpg`
  - 例: `note_4koma_meeting-ai_2026-03-11.jpg`
- 1記事あたり上限: サムネイル1枚 + 図解3枚 + 4コマ漫画2枚
