# note 記事エディタ

## アクセス方法

1. `/notes/new` にアクセス → `editor.note.com/new` にリダイレクト
2. 自動的に下書きが作成され `editor.note.com/notes/<id>/edit/` に遷移
3. ページ読み込みに時間がかかる場合がある（3秒待機推奨）

## エディタ構造

### ヘッダーナビ

| 要素 | ref例 | 操作 |
|------|-------|------|
| 閉じる | e21 | エディタを閉じる（beforeunloadダイアログあり） |
| その他 | e27 | 追加メニュー |
| 下書き保存 | e33 | 下書きとして保存 |
| 公開に進む | e36 | 公開設定画面へ |

### 文字カウンター

- 左上に「0 文字」表示（complementary領域）

### メイン編集エリア

| 要素 | ref例 | 型 | 備考 |
|------|-------|----|------|
| 画像を追加 | e41 | button | 見出し画像（サムネイル） |
| 記事タイトル | e47 | textbox | `textbox "記事タイトル"` |
| 本文エリア | e51 | textbox (active) | ブロックエディタ（contenteditable） |
| メニューを開く | e55 | button | ブロック追加メニュー（+ボタン） |

### 本文入力の注意

**ブロックエディタ方式**:
- 本文は `contenteditable` ベースのブロックエディタ
- `fill_form` は使用不可。`browser_click` でフォーカス → `browser_type` で入力
- 段落ごとにEnterで新しいブロックが生成される
- 「メニューを開く」ボタンから画像・埋め込み等のブロックを追加可能

### 操作手順（記事作成）

```
1. navigate → https://note.com/notes/new
2. wait_for 3秒（リダイレクト＋ロード待ち）
3. click ref="画像を追加" → サムネイル設定（任意）
4. click ref="記事タイトル" → type でタイトル入力
5. click ref="本文エリア" → type で本文入力（改行はEnter）
6. click ref="公開に進む" → 公開設定画面へ
```

## 公開設定画面

URL: `editor.note.com/notes/<id>/publish/`
「公開に進む」クリック後に遷移。タイトル＋本文が空だとバリデーションエラー（ダイアログ表示）。

| 項目 | ref例 |

| ハッシュタグ | e97 |

### ハッシュタグ

- `combobox "ハッシュタグを追加する"` — テキスト入力で追加
- 本文から自動候補が表示される（例: `#本文`, `#テスト`）
- お題/コンテストへの参加ボタンが一覧表示される

### 記事タイプ

| 選択肢 | radio |
|--------|-------|
| 無料 | checked（デフォルト） |
| 有料 | — |

### 記事の追加

- **マガジン**: checkbox（デフォルトchecked）
- **メンバーシップ**: checkbox
- マガジン一覧から追加ボタンで選択

### 詳細設定

| 設定項目 | 型 | デフォルト | 備考 |
|---------|-----|----------|------|
| クリエイターページに表示 | switch | ON | |
| AI学習対価還元プログラム | switch | ON | |
| コメントの受けつけ | switch | ON | **プレミアム機能** |
| 予約投稿 | button「日時の設定」 | — | **プレミアム機能** |

### 予約投稿（プレミアム限定）

- 「日時の設定」ボタンをクリックで日時選択UIが開く（未探索 — プレミアム契約が必要）
- 無料プランでは利用不可

### 操作手順（公開設定 → 投稿）

```
1. ハッシュタグを combobox に入力
2. 記事タイプを選択（通常は「無料」のまま）
3. マガジン追加（必要に応じて）
4. 詳細設定を確認
5. 「投稿する」をクリック → 公開
```

## 既知の問題

- エディタ離脱時に beforeunload ダイアログが必ず発火する
- `browser_handle_dialog(accept=true)` で処理必須
