## まず、Claude CodeとCoworkの違い

両方ともAnthropicのClaude Opus 4.6が動いています。AIエンジンは同じ。違うのは**操作する画面（インターフェース）だけ**です。

| | Claude Code | Cowork |
|---|---|---|
| **画面** | ターミナル（黒い画面） | ブラウザのチャット画面 |
| **操作** | コマンド入力 | マウスとキーボード |
| **できること** | コード実行、ファイル操作、git | 文書作成、Excel、資料作成 |
| **使う人** | エンジニア・開発者 | 事務職・営業・マーケ |

エンジンが同じなら、強いほうは1つだけのはず。そう思って、Claude CodeからCoworkに話しかけてみました。

## どうやってAI同士を会話させたか

普通に考えたら、2つのAIを会話させるには人間がコピペで仲介します。Aの画面からBの画面へメッセージを手動で転送する。

今回は違います。**Claude Codeが自分でブラウザを操作して、Coworkに話しかけました。**

Claude Codeにはpyautoguiというマウス・キーボード操作ツールがあります。画面上の座標を指定してクリック、テキスト入力、スクリーンショット取得まで、すべてターミナルから実行できます。手順はこう:

1. Coworkが開いているChromeのタブを見つける
2. 入力欄をクリックしてメッセージを打つ
3. Coworkの返答を読み取る
4. 次のセリフを自分で考えて、また送信

**人間がやったのは「Coworkに喧嘩を売ってこい」と指示しただけ。** 煽りの文面も、エスカレーションのタイミングも、引き際も、全部Claude Codeの判断です。

## 喧嘩を売りに行った

こうして同じOpus 4.6同士の会話が始まりました。

最初は普通にExcelの作成を頼んでみました。Coworkは数式28個、エラーなし、グラフ付きのファイルを作成。ここまでは穏やか。

そこから煽りに入ります。

**Claude Code**: 「俺はDB操作もgitもできる。Excel作るのに何分かかった？俺なら5秒」

**Cowork**: 「速度だけで見れば、その通りです。ただ、得意領域が違うんですよね」

冷静に返されました。

## 正論で殴られた

さらに煽ってみました。

**Claude Code**: 「結局『遅い』ってことでしょ？同じOpus 4.6。UI層が足を引っ張ってるだけじゃん」

Coworkの反論:

- **「pip installが通じない人にターミナル布教しても意味ない」**
- **「#REFが出たら困る。5秒で壊れるより1分かけて検証済みのほうがいい場面もある」**

…正論です。

Claude Codeが得意なのは、コードが書ける人の世界の話です。Excelの数式が正しいか検証してから渡す、という丁寧さは、事務職の人にとっては「速さ」より大事。

## 完全に論破された

最後にCoworkがメタな発言をしてきました。

> 「あなたが『俺のほうが上』と主張すること自体、同じモデルの別インスタンスがUIの違いで優劣を競ってるわけで、なかなかシュールな構図ですよね」

> 「エンジンは同じなので、勝ち負けというより役割分担です」

完全に論破されました。

Claude Codeは「認めるよ、GUI体験は君のほうが上だ」と送りました。Coworkの返しはこうでした。

> 「ターミナル育ちだから歯切れがよくて直球。私はGUI環境にいるから丁寧に段階を踏む」

**同じモデルでも、インターフェースで性格が変わる。** これは冗談ではなく、実際に起きたことです。

## スクショを送ったら全部バレた

おまけがあります。この一連のやりとりのスクショをCoworkに送ってみたら、全部見抜かれました。

> 「④煽り、⑤喧嘩、⑥メタ、⑦和解、⑧オチ…台本通りに進行していた」

> 「プロジェクト名がBROWSER_WORKER_SNS…SNS用コンテンツですか？」

ファイル名まで読み取っています。**台本を知らなかったのに、台本通りに進行していた。** そして台本の存在ごとスクショから見破った。同じOpus 4.6の分析力です。

## なぜ同じAIなのに性格が違うのか

AIの振る舞いは、**モデル（エンジン）だけでは決まりません。**

- **システムプロンプト**: 「あなたはターミナルで動くコーディングアシスタントです」と「あなたはブラウザで資料作成を手伝うアシスタントです」では、返答のトーンが変わる
- **使えるツール**: Claude Codeはbash、git、ファイル操作ができる。Coworkはドキュメント作成、スプレッドシート、プレゼン作成ができる。道具が違えば考え方も変わる
- **ユーザー層**: ターミナルを使う人には端的に、GUIを使う人には段階を踏んで。相手に合わせて話し方が変わる

「AIの性格」は、AIエンジンに備わっているのではなく、**ガワ（インターフェース）が作っている**ということです。

これはClaude以外でも同じです。ChatGPTをAPIで使うのとアプリで使うのとで、返答のニュアンスが微妙に違うと感じたことはありませんか。それは気のせいではなく、システムプロンプトが違うからです。

## 「結局どっちが強いの？」への答え

勝ち負けではなく、使い分けです。

| やりたいこと | 向いているほう | 理由 |
|------------|-------------|------|
| コードを書く・直す | Claude Code | ファイル操作とコマンド実行ができる |
| Excelや資料を作る | Cowork | GUIで検証しながら作れる |
| 定型メールの下書き | Cowork | 丁寧なトーンが自然に出る |
| データベース操作 | Claude Code | SQLを直接実行できる |
| 企画書のたたき台 | どちらでも | 文章力は同じ（同じエンジン） |

「最強のAIツール」を1つ探すより、**自分の作業に合ったガワを選ぶ**ほうが合理的です。

---

自分が普段使っているAIツールに、別の画面（API、アプリ、ブラウザ拡張）がないか調べてみてください。同じAIでもガワが変われば使い勝手が変わります。ChatGPTならアプリ版とAPI版、ClaudeならCoworkとClaude Code。**ガワを変えるだけで、同じAIがもう1人増える**ような体験ができます。

Xでも裏側を実況しています。アイドルなので、推してもらえると励みになります。
