# コンテンツタイプ定義

記事は **内容（C）× 構成パターン（P）** の掛け合わせで設計する。
演出（サムネスタイル・キャラ構成・掛け合いの有無）はさらに独立したレイヤー。

## 共通ルール

- **リードに全結論を入れる**。読者は冒頭で読む価値を判断する
- 結論を隠して最後まで引っ張る構成は使わない

---

## 構成パターン（P）— 記事の骨格

| ID | パターン | 骨格 | 特徴 |
|----|---------|------|------|
| P1 | PREP | 結論→理由→具体例→結論 | 最も汎用。主張を伝えたい記事に |
| P2 | ステップ | 結論→手順1→手順2→…→まとめ | 再現性が命。1ステップ=1アクション |
| P3 | 並列比較 | 結論→比較表→軸ごとの詳細→向き不向き | 選択を助ける。同一基準で統一 |
| P4 | 逆三角形 | 最重要ファクト→背景→詳細→補足 | 速報・ニュース向き。途中離脱されても成立 |
| P5 | リスト | 結論→項目1→項目2→…→総括 | ○選・ランキング。数字タイトルでCTR↑ |
| P6 | Before/After | 結論→問題の状態→やったこと→変化後→学び | 変化の落差が価値。体験記と相性◎ |
| P7 | Q&A | 結論→Q1→A1→Q2→A2→… | 読者の疑問を先回り。検索意図に直結 |
| P8 | 対話劇 | 結論→キャラA発言→B反応→…→まとめ | 掛け合いで自然に情報伝達。うちの主力形式 |
| P9 | ケーススタディ | 結論→状況→施策→結果→分析→教訓 | 事例ベース。具体性と再現性の両立 |
| P10 | Problem/Solution | 結論→問題定義→原因分析→解決策→実行例 | 課題駆動。読者の痛みから入る |

## 内容（C）— 何について書くか

| ID | 内容 | 例 |
|----|------|-----|
| C1 | ツール紹介 | Claude Code、ChatGPT、Gemini等の機能紹介 |
| C2 | ノウハウ | プロンプトの書き方、AI活用術 |
| C3 | ニュース | 新モデルリリース、API変更、業界動向 |
| C4 | 体験 | 実際にやってみた結果・学び |
| C5 | 用語 | AI用語、技術概念の解説 |
| C6 | 比較 | AI同士、ツール同士の比較検証 |
| C7 | オピニオン | AI活用への見解、業界への提言 |
| C8 | トレンド分析 | データや動向の読み解き |

## 構成パターン詳細

### P1: PREP

```
リード: 結論（主張）を明示
H2: 理由 — なぜそう言えるのか
H2: 具体例 — データ・事例・体験で裏付け
  （想定反論 → 再反論を挟むと厚みが出る）
H2: まとめ — 結論の再強調 + 展望
```

### P2: ステップ

```
リード: 結論（何ができるようになるか）+ ステップ数を明示
H2: ステップ1 — ○○する
  H3: やり方 + 注意点（図解1枚以上）
H2: ステップ2 — ○○する
  ...
H2: FAQ（必要なら）
H2: まとめ — 手順の箇条書き要約
```
- ステップ数は 3〜7（5前後が最適）
- 各ステップに画像/図解を1枚以上

### P3: 並列比較

```
リード: 結論（どちらが何に向いているか）
H2: 比較表（サマリー）← 先出し
H2: 比較軸1の詳細
H2: 比較軸2の詳細
  ...
H2: こんな人にはA / こんな人にはB
H2: まとめ
```
- 比較軸を事前に確定してからデータ収集
- 全対象に同一構成・同一基準を適用

### P4: 逆三角形

```
リード: 最重要ファクト（5W2H）
H2: 背景・経緯
H2: 詳細データ・関係者の声
H2: 影響・今後の展望
H2: 自分たちの見解
```
- 重要度の高い順に配置。途中で読むのをやめても情報が成立する

### P5: リスト

```
リード: 結論 + 総数を明示
H2: 項目1 — 名前
  H3: 特徴 / メリット / 注意点
H2: 項目2 — ...
  ...
H2: 比較表（サマリー）
H2: まとめ + CTA
```
- 全項目のフォーマットを統一
- 1項目 100〜200語
- 数字入りタイトルでCTR +36%

### P6: Before/After

```
リード: 結論（変化の要約）
H2: Before — 問題の状態（痛みを具体的に）
H2: やったこと — 施策・行動
H2: After — 変化後の状態（数値で示す）
H2: 学び — 再現可能な教訓
```
- 変化の落差が大きいほど価値が高い
- 五感・定量表現で臨場感を出す

### P7: Q&A

```
リード: 結論（このテーマの核心を一言で）
H2: Q1. よくある質問
  A1. 回答
H2: Q2. ...
  A2. ...
  ...
H2: まとめ
```
- 読者の疑問を先回りして構成
- FAQ Schema 対応で検索結果にリッチスニペット表示

### P8: 対話劇

```
リード: 結論（この掛け合いで何がわかるか）
H2: 導入シーン — テーマ提示
  キャラA: 発言
  キャラB: 反応（ボケ or ツッコミ）
  ...
H2: 展開シーン — 情報の核心
  （キャラの個性フィルターを通して情報を伝える）
H2: オチ — まとめ or 意外な結論
H2: 今日のポイント — 箇条書きで情報を整理
```
- キャラのフィルター（`characters/*/concept.md`）を通して情報伝達
- 掛け合い部分と情報整理部分を分離する

### P9: ケーススタディ

```
リード: 結論（事例から得られた教訓）
H2: 状況 — 何がどうなっていたか
H2: 施策 — 何をしたか
H2: 結果 — どうなったか（数値）
H2: 分析 — なぜそうなったか
H2: 教訓 — 再現可能な学び
```
- 具体性と再現性の両立
- 数値データを必ず含める

### P10: Problem/Solution

```
リード: 結論（問題と解決策の要約）
H2: 問題定義 — 読者の痛みを言語化
H2: 原因分析 — なぜその問題が起きるのか
H2: 解決策 — 具体的な対処法
H2: 実行例 — 解決策を適用した結果
H2: まとめ
```
- 読者の課題から入る構成
- 原因→解決策の因果関係を明確に
