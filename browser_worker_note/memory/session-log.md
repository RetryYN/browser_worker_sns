### 発見（次の探索者へ）
- **X 4画像グリッド**: 1枚を分割する方式はXの自動ギャップで必ず破綻する。独立生成が唯一の実用的方法
- **ピクセルアート暴走防止**: `8-bit style` + `limited palette` + `coarse large pixels` + `no anti-aliasing` の4点セットが必須
- **日本語テキスト**: 2-4文字が限界。長文は化ける。枠付きバナー内配置 + 引用符囲みが効く
- **図解のサブタイプ**: フロー/比較/リスト/概念図の個別定義は不要。シンプルなものはデフォルト、複雑なものはボード型の2択で十分
- **generate_image.py の Windows 対応**: cp932 で em-dash 等が出力できない。sys.stdout.reconfigure 必須

### 改善すべきこと

### 発見（次の探索者へ）
- **クイズは正方形(1:1)が最適**: 横長だとクロップで上下が切れる。Xタイムラインでも正方形は目立つ
- **CTA文言は各タイプ別に変える**: 選択式→「コメントで回答!」、○×→「○か×でリプライ!」等。汎用CTAより反応が良い
- **間違い探しは1枚内左右分割**: 2枚の独立生成ではなく1枚の中に左右で描かせる。ラベル(左/右)必須
- **`--quiz-data`は選択式・ランキングのみ**: ○×、穴埋め、間違い探し等はプロンプト本文だけで十分
- **図解のデフォルトとボード型は両方安定**: フロー図3ステップ(デフォルト)、比較表(ボード型)ともにキャラ配置・テキスト・レイアウト問題なし
