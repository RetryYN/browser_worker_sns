# generate-content note ワークフロー

`generate-content.md` の Phase 2（執筆）完了後、note 固有の検証・画像生成・校正・公開を実行する。

**前提**: 下書きファイル `knowledge/drafts/note_{スラッグ}.md` が存在し、ステータスが「下書き」であること。

---

## Phase 2.5: 検証（メインコンテキスト）

### 2.5-1. ファクトチェック

下書き内の事実・数値を一次ソースと突合する。

```
チェック項目:
- [ ] 数値データ: 一次ソースと完全一致するか
- [ ] 固有名詞: 正式名称か（製品名・企業名・人名）
- [ ] 日付・時期: 正確か（「最近」等の曖昧表現がないか）
- [ ] URL: リンク先が存在し、内容が記事の記述と一致するか
- [ ] 引用: 原文の意味を歪めていないか
```

### 2.5-2. ソーススクリーンショット

実演型記事（C1, C4）の場合、実際の画面スクリーンショットを取得する。

- 実演可否は `/research` の Step 8 で判定済み
- 実演可 → Playwright で操作画面を撮影し `knowledge/data/images/` に保存
- 実演不可 → 図解で代替（2.5-5 で生成）

### 2.5-3. 具体性チェック

```
チェック項目:
- [ ] 抽象的な主張に具体例（画面/数値/手順）が伴っているか
- [ ] 「すごい」「便利」で終わっていないか（→ 何がどう便利かを記述）
- [ ] Before/After がある場合、After だけでなく Before の具体も示しているか
- [ ] 読者が記事を読んだ後に再現できるステップがあるか
```

### 2.5-4. ロジック検証（本文レベル）

Phase 1-6b のアウトラインレベルのロジック検証を、執筆後の本文でもう一度通す。

```
チェック項目:
- [ ] セクション間の論理接続: 前の結論 → 次の前提が繋がっているか
- [ ] 主張と根拠の対応: 各主張に裏付けがあるか
- [ ] リードの約束: リードで言った結論が本文で証明されているか
- [ ] スコープ: タイトルの射程を超えた脱線がないか
```

### 2.5-5. 画像生成

note記事に必要な画像を生成する。

#### 画像種別と仕様

| 種別 | サイズ | 用途 | 命名規則 |
|------|--------|------|---------|
| サムネイル | 1280×670 (1.91:1) | 記事トップ・OGP | `note_thumbnail_{style}_{slug}_{date}.jpg` |
| 図解 | 1536×1024 | 本文中の概念図・比較図 | `note_diagram_{slug}_{date}.jpg` |
| ソーススクショ | 任意 | 実画面の証拠 | `source_{slug}_{連番}.png` |

#### サムネイル生成コマンド

```bash
python scripts/generate_image.py thumbnail "<テーマを表現するプロンプト>" \
  --topic "<テーマ>" --layout blackboard [--chars ...]
```

**サムネイルルール（F13）:**
- テキスト必須。装飾的イラストだけはNG
- 黒板スタイル（`--layout blackboard`）が標準
- セーフゾーン: 上下110px はクロップされる可能性あり → 重要要素は中央に配置

#### 図解生成コマンド

```bash
python scripts/generate_image.py diagram "<図解の内容を説明するプロンプト>" \
  --topic "<テーマ>" [--chars ...]
```

### 2.5-6. 画像検証

生成した全画像を検証する。

```
チェック項目:
- [ ] サムネイル: テキストが読めるか（文字化け・切れなし）
- [ ] サムネイル: セーフゾーン内に重要要素が収まっているか
- [ ] 図解: 情報が正確か（数値・ラベル・矢印の向き）
- [ ] 図解: 本文の説明と矛盾していないか
- [ ] ソーススクショ: 個人情報・機密情報が映り込んでいないか
- [ ] ファイル名: 命名規則に沿っているか
```

### 2.5-7. 検証完了条件

以下をすべて満たしたら、下書きのステータスを「検証済み」に更新する。

```
- [ ] ファクトチェック: 全項目 ✅
- [ ] 具体性チェック: 全項目 ✅
- [ ] ロジック検証: 全項目 ✅
- [ ] 画像: 全画像生成済み + 検証 ✅
- [ ] 下書きの画像マニフェスト: 全画像のパスと挿入位置を記録済み
```

下書きファイルの画像セクションを更新:

```markdown
## 画像

| # | 種別 | ファイル | 挿入位置 | 検証 |
|---|------|---------|---------|------|
| 1 | サムネイル | `knowledge/data/images/note_thumbnail_blackboard_{slug}_{date}.jpg` | 記事トップ | ✅ |
| 2 | 図解 | `knowledge/data/images/note_diagram_{slug}_{date}.jpg` | H2-2 直後 | ✅ |
```

## Phase 3.5: note 文字数・フォーマットチェック

校正完了後、note 固有の制約をチェックする。

```
チェック項目:
- [ ] タイトル: 80文字以内（note上限）
- [ ] 本文: 3,000〜8,000文字（推奨レンジ。長文がnoteの強み）
- [ ] H2見出し数: 3〜6個（多すぎると目次が長くなる）
- [ ] 画像枚数: 3〜8枚（サムネ含む）
- [ ] ハッシュタグ: 3〜5個（多すぎると散漫）
- [ ] 参考リンク: 各ソースの直下にインライン配置（末尾まとめNG = F12）
- [ ] 太字: キーワードや結論に適切に使用（テキストの壁にしない = F16 formatting）
```

チェック完了 → 下書きステータスを「校正済み」に更新。

## Phase 4: note 公開

### 4-1. 公開前の最終確認

ユーザーに以下を提示して承認を得る:

```
■ 公開確認
- タイトル: {タイトル}（{N}文字）
- C×P×T: {C×P×T}
- 本文: {N}文字
- 画像: {N}枚
- ハッシュタグ: #{tag1} #{tag2} ...
- 下書き: knowledge/drafts/note_{スラッグ}.md

→ 公開してよいですか？
```

### 4-2. note エディタへの転記

`/run-task note-article-publish` を使用するか、以下の手順で手動実行。

#### エディタ操作手順

```
1. navigate → https://note.com/notes/new
2. wait_for 3秒（リダイレクト＋ロード待ち）
3. サムネイル設定:
   - 「画像を追加」→ file_upload でサムネイル画像
4. タイトル入力:
   - 「記事タイトル」textbox → type
5. 本文入力（DOM一括操作）:
   - browser_evaluate で ProseMirror innerHTML に HTML を挿入
   - dispatchEvent(new Event('input', { bubbles: true }))
6. 画像挿入（各画像ごと）:
   - 空段落をクリック → +メニュー → 画像 → file_upload
   ⚠️ テキスト段落の途中で画像挿入しない（分断される）
7. テキスト装飾:
   - browser_evaluate で <strong> タグ追加
8. 目次挿入:
   - リード直後の空段落 → +メニュー → 目次
```

#### エディタ上検証

```
チェック項目:
- [ ] 見出し（H2/H3）: 順序・テキストが下書きと一致
- [ ] 目次: 挿入済み・見出しと対応
- [ ] 段落・改行: DOM操作で崩れていないか
- [ ] サムネイル: エディタ上部に表示
- [ ] 本文中画像: 全画像が正しい位置・順序
- [ ] 太字: 意図した箇所のみ
- [ ] リンク: 参考リンクが正しいURL
- [ ] 参考リンク配置: ソース下にインライン
```

### 4-3. 公開設定

```
1. 「公開に進む」→ publish 画面に遷移
2. ハッシュタグ: combobox に slowly:true で入力 → ドロップダウンから選択
3. 記事タイプ: 無料（デフォルト）
4. 詳細設定確認:
   - クリエイターページに表示: ON
   - AI学習対価還元: ON
   - コメント受付: ON
5. ★ ユーザー最終承認 ★
6. 「投稿する」クリック
7. スキ設定: スキップ or 設定
```

### 4-4. 公開後検証

公開記事ページ（`https://note.com/akatu_unison/n/<記事ID>`）にアクセスし検証。

| カテゴリ | チェック項目 |
|---------|------------|
| 構造 | タイトル一致 / H2H3順序 / 目次 / 段落改行 |
| 画像 | サムネ表示 / 本文中画像位置 / 画像順序 |
| テキスト | 太字 / リンク / 参考リンク配置 / 文字化け |
| メタ | ハッシュタグ / 記事URL |

### 4-5. 記録

1. 下書きステータスを「投稿済み」に更新（公開URLを追記）
2. `memory/session-log.md` に記録
3. `note_analytics.py` に登録（存在する場合）:
   ```bash
   python scripts/note_analytics.py add --article-id <id> --date YYYY-MM-DD \
     --title "タイトル" --content-type C分類 --treatment T軸 \
     --chars 文字数 --images 画像枚数 --thumb-style blackboard \
     --hashtags "tag1,tag2" --url "https://note.com/akatu_unison/n/<id>" \
     --draft "knowledge/drafts/note_{slug}.md"
   ```
