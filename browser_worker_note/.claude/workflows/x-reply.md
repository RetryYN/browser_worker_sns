# /x-reply ワークフロー

ブラウザでX内検索 → 投稿を精査 → リプ作成・投稿。1日10リプ目安。

## 使い方

```
/x-reply              ← 検索からリプまで一気通貫
/x-reply <URL>        ← 特定の投稿に単発リプ
```

## 起動時に読むファイル

1. `knowledge/sites/x/reply-strategy.md` — ターゲット・検索ワード・トーン・NG
2. `knowledge/account-concept.md` — ペルソナ・トーン
3. `knowledge/sites/x/index.md` — ブラウザ操作フロー

## フロー

### ① 検索（ブラウザ）

X内の検索機能を使ってターゲット投稿を探す。

1. `x.com/search` に navigate
2. `reply-strategy.md` のキーワード辞書から検索ワードを入力
3. 「最新」タブに切り替え（直近の投稿を優先）
4. 検索結果を snapshot でスクロールしながら確認

キーワードは3層（就活生/理系学生/大アカ向け）からローテーション。

**検索のコツ:** 正式キーワード（「就活 AI ChatGPT」等）はステマばかりヒットする。スラング（「チャッピー」「GPT」等）で検索すると本物の学生が見つかりやすい。

### ② 精査・選別

検索結果の投稿を1件ずつ見て、リプするか判断する。

**★ プロフ確認必須（就活生・理系学生層）:**
投稿が良さそうでも、まずプロフを見る。以下があれば本物の学生:
- 志望業界（「メーカー志望」「IT志望」等）
- 大学名 or 学部
- 卒年（「26卒」「27卒」等）

企業アカウント・人事・ステマ（「AI副業」「フォロバ100%」等）は即スキップ。

**リプする:**
- AIに前向きな投稿
- 質問・困り事系
- 大アカ（インプ狙い）でリプが埋もれない程度のもの
- 「やってみた」報告

**スキップ:**
- AI否定的・懐疑的
- 炎上中・論争中
- 情報商材・スクール系・企業アカウント
- リプが多すぎて埋もれる

良さそうな投稿を見つけたら、その投稿をクリックして全文を読む。

### ③ リプ作成・投稿

1. 投稿の内容に合ったリプ本文を生成（100字以内）
2. トーン: 共感 / 補足情報 / 応援 / 実体験シェア（`reply-strategy.md` 参照）
3. ★ ユーザーに本文を提示して承認を得る
4. リプ入力欄にテキスト入力 → 返信ボタンをクリック
5. snapshot で送信確認
6. 次の投稿へ（検索結果に戻るか、次のキーワードで検索）

**ルール:**
- URL貼らない、誘導しない、宣伝しない
- 「いいですね！」だけの中身のないリプはしない
- 同じアカウントに1日2リプ以上しない
- 画像は原則つけない（テキストリプのみ）

---

## 単発リプ（URL指定）

```
/x-reply https://x.com/user/status/1234567890
```

1. 指定URLに navigate → 投稿全文を読む
2. Go/NG判定
3. リプ本文生成 → ★ ユーザー承認 → 投稿

---

## エスカレーション

- リプ投稿の直前に**必ずユーザー承認**
- Go/NG判断に迷ったら → Skip（迷ったらやらない）
- ネガティブなリプ返しが来た場合 → **反論しない。無視する**
