# /x-reply ワークフロー

ブラウザでX内検索 → 投稿を精査 → リプ作成・投稿。1日10リプ目安。

## 使い方

```
/x-reply              ← 検索からリプまで一気通貫
/x-reply 10           ← 10件リプを回す
/x-reply <URL>        ← 特定の投稿に単発リプ
```

---

## 起動時に読むファイル

1. `knowledge/sites/x/reply-strategy.md` — ターゲット・検索ワード・トーン・NG・Few-shot
2. `knowledge/account-concept.md` — ペルソナ・トーン
3. `knowledge/sites/x/index.md` — ブラウザ操作フロー

## フロー

### ① 検索（ブラウザ）

X内の検索機能を使ってターゲット投稿を探す。

1. `x.com/search?q=<キーワード>&src=typed_query&f=top` に navigate（URLで直接指定）
2. 5秒待機（SPA描画待ち）
3. snapshot で結果確認

**検索タブの使い分け:**

| 層 | 検索タブ | 理由 |
|----|---------|------|
| 大アカ寄生 | **話題のポスト**（`f=top`） | エンゲージメント順。寄生先として有効な投稿が上に来る |
| 就活生・理系 | **最新**（`f=live`） | 直近の投稿を拾う。鮮度が重要 |

**キーワード:** `reply-strategy.md` の検索キーワード辞書から選択。`lang:ja` を付けて日本語に絞る。

**スクロール:** End キーで下まで読み込み → snapshot で新しい候補を確認。

### ② 精査・選別（2段階フィルタ）

候補を見つけたら **プロフ確認 → 投稿内容確認** の2段階で判定。

#### Stage 1: プロフ確認（navigate してプロフを見る）

**即NG:**
- 情報商材・オンラインスクール系（「AI副業」「公式LINE」「セミナー」「講座」）
- Gemini / NotebookLM 専門家（≒情報商材）
- マウント系（「CTO経験もある自分は〜」的な上から目線）
- セミナー販売系
- 驚き屋スタイル（「ヤバい！！」「神アプデ！」が口癖）
- 企業アカウント・人事/HR系
- 既にリプ済みのアカウント

**Go判定:**
- 本物の技術者（GitHub、実プロダクト、具体的な技術スタック）
- 就活生（卒年、大学名、志望業界がプロフにある）
- AI活用を実践している個人

#### Stage 2: 投稿内容確認

**NG:**
- リプ文に企業名・サービス名が必要になる投稿 → パス
- 闇深いジャンル（勤怠、労務、給与等）
- 炎上中・論争中
- 24時間以上前の投稿（鮮度切れ）

**Go:** 上記に該当しなければ Go。

#### ユーザーが却下した場合

「パス」「NG」等のフィードバックがあったら即次の候補へ。理由を記憶して同じタイプを繰り返さない。

### ③ リプ作成

1. 対象投稿の全文を読む
2. `reply-strategy.md` のFew-shotを参考にリプ案を作成
3. **1文が正解。最大2文。** 「とかでいいよ」が正解ライン
4. ★ ユーザーに本文を提示して承認を得る

**リプ作成のルール:**
- URLを貼らない、誘導しない、宣伝しない
- 「弊社も」「当社では」は絶対NG
- 経験を捏造しない（やっていないことを「やってみた」と書かない）
- 「知らなかった」「よさそう」等の正直な感想はOK
- 元投稿のトーンに合わせる（ネタにはネタで、真面目には真面目に）

### ④ 投稿

1. 対象投稿ページに navigate → 5秒待機
2. 返信テキストボックスをクリック
3. `type` で入力（`slowly: true`）
4. 「返信」ボタンをクリック
5. snapshot でトースト確認（「ポストを送信しました」+ reply URL）
6. reply URL から reply_id を取得

### ⑤ DB記録

投稿成功後、即座にDBに記録:

```bash
python scripts/x_analytics.py reply-add \
  --reply-id {reply_id} \
  --date {YYYY-MM-DD} \
  --time "{HH:MM}" \
  --target-user {handle} \
  --target-post-id {post_id} \
  --target-url {url} \
  --layer big_account \
  --pattern "{パターン名}" \
  --text "{リプ本文}" \
  --url "{リプURL}"
```

### ⑥ 次の候補へ

検索結果に戻り（ブラウザバックまたは再検索）、次の候補を探す。

## 単発リプ（URL指定）

```
/x-reply https://x.com/user/status/1234567890
```

1. 指定URLに navigate → 投稿全文を読む
2. プロフ確認 → Go/NG判定
3. リプ本文生成 → ★ ユーザー承認 → 投稿 → DB記録

---

## エスカレーション

- リプ投稿の直前に**必ずユーザー承認**
- Go/NG判断に迷ったら → Skip（迷ったらやらない）
- ネガティブなリプ返しが来た場合 → **反論しない。無視する**
